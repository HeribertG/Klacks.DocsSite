# Administración de usuarios

En Configuración > Administración de usuarios se gestionan las **cuentas de acceso** (usuarios del sistema) con las que las personas inician sesión en Klacks — esto no es lo mismo que la gestión de empleados o de direcciones, ya que una cuenta de acceso puede, pero no tiene por qué, estar vinculada a una ficha de datos maestros de una persona.

## Cómo funciona

**Añadir usuario:** El diálogo "+ Añadir usuario" genera automáticamente, a partir del nombre y apellido (mínimo 2 caracteres cada uno), una propuesta de nombre de usuario. La dirección de correo electrónico es obligatoria y debe contener un carácter @; solo se puede guardar cuando todos los campos están correctamente rellenados. Al crear la cuenta se genera automáticamente una contraseña — para que la persona pueda establecer una propia, utilice después "Restablecer contraseña" en su fila.

**Editar usuario:** Un doble clic en la fila del nombre abre el diálogo de edición. Allí se puede modificar posteriormente el nombre de usuario — en la creación es solo una propuesta.

**Restablecer contraseña:** El enlace "Restablecer contraseña" en la fila de un usuario envía un correo electrónico a la dirección registrada, mediante el cual la persona establece una nueva contraseña por sí misma.

**Eliminar usuario:** El botón de papelera elimina definitivamente una cuenta de acceso, tras confirmación en el diálogo de eliminación.

**Asignar permisos:** Mediante un menú desplegable en la fila se asigna **Supervisor** o **Admin**; el cambio surte efecto de inmediato, sin necesidad de un clic de guardado separado. Lo que estos dos roles pueden hacer en detalle se describe en "Roles y permisos" en Primeros pasos — aquí solo se trata de la gestión de las propias cuentas.

## Bueno saber

- El usuario actualmente conectado también se muestra en la lista, pero no puede editarse a sí mismo, cambiar sus propios permisos ni eliminarse allí.
- Una cuenta de acceso es independiente de la gestión de personas/empleados — por ejemplo, se puede crear una cuenta para una empresa de TI externa sin necesidad de crear una ficha de empleado para ello.
- No existe un establecimiento directo de contraseña por parte del administrador: las cuentas nuevas reciben una contraseña generada automáticamente, y cualquier cambio posterior se realiza mediante el enlace de correo electrónico "Restablecer contraseña".
