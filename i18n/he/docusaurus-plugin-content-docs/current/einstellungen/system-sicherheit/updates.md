# עדכונים

תחת הגדרות > עדכונים ניתן לראות את הגרסה המותקנת כעת ואת הגרסה העדכנית ביותר הזמינה של Klacks, לשלוט בהתנהגות העדכון האוטומטי ולהפעיל ידנית, במידת הצורך, עדכון או שחזור לגרסה קודמת (Rollback).

## איך זה עובד

**סטטוס:** הכרטיסייה מציגה את הגרסה הנוכחית, האם קיימת גרסה חדשה יותר, וכן את פעולת העדכון הפועלת כרגע או שבוצעה לאחרונה יחד עם הסטטוס שלה.

**הפעלת עדכון או שחזור:** דרך "התקן עכשיו" מפעילים ידנית עדכון זמין; "בטל" מבטל את פעולת העדכון האחרונה שהצליחה. שתי הפעולות נעולות כל עוד פעולה אחרת כבר פועלת.

**הגדרת האוטומציה:**
- **הפעלת עדכונים אוטומטיים** מפעילה או מכבה את בדיקת העדכון האוטומטית ואת ביצועה.
- **התראה בלבד** לא מפעילה עדכון אוטומטי, אך מודיעה כאשר גרסה חדשה זמינה.
- **ערוץ** בוחר בין `Stable` ל-`Beta` כמקור העדכונים.
- **מרווח בדיקה** (שעות) ו**חלון תחזוקה** (שעת התחלה/סיום) קובעים באיזו תדירות ובאיזה חלון זמן מתבצעת הבדיקה או ההתקנה.
- **מספר גיבויים לשמירה** קובע כמה גיבויים נשמרים לפני מיגרציה, לפני שהישנים יותר נמחקים אוטומטית.

**היסטוריה:** טבלה מציגה את פעולות העדכון האחרונות עם סוג, גרסת יעד, סטטוס ומועד הבקשה.

## כדאי לדעת

- לפני כל מיגרציה, Klacks יוצרת אוטומטית גיבוי - ללא גיבוי לא מתבצעת שום מיגרציה.
- אם בדיקת התקינות נכשלת לאחר עדכון, Klacks חוזרת אוטומטית לגרסה הקודמת (שחזור גיבוי → הפעלת הגרסה הישנה → בדיקה חוזרת); רק אם השחזור עצמו נכשל, נדרשת התערבות ידנית.
- שחזור ידני (Rollback) משחזר באופן ממוקד את הגיבוי השייך לפעולה המוצלחת האחרונה ומפעיל את גרסת המוצא שלו - לא סתם "חזרה גרסה אחת אחורה".
- עדכון ושחזור הן פעולות המיועדות למנהלים (Admin) בלבד.
