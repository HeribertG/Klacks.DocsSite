# קלט ופלט קולי

ניתן להפעיל את Klacksy גם בקול — תחת הגדרות > קול Klacksy אתם קובעים באיזה שירות הקול שלכם מומר לטקסט (Speech-to-Text) ובאיזה קול Klacksy עונה (Text-to-Speech).

## איך זה עובד

**זיהוי דיבור (STT):** ארבעה מנועים זמינים לבחירה — הדפדפן החינמי (Web Speech API), Deepgram (הזרמה בזמן אמת), Groq Whisper (מהיר וזול) וכן AssemblyAI (הזרמה). עבור כל האפשרויות מלבד הדפדפן, אתם מגדירים מפתח API משלכם וניתן לבדוק את החיבור ישירות.

**פלט קולי (TTS):** גם כאן ארבעה ספקים — Edge TTS (חינמי), OpenAI TTS, ElevenLabs ו-Google Cloud TTS. בהתאם לספק שנבחר, Klacks טוען את הקולות הזמינים אוטומטית; גם כאן ניתן לבדוק את החיבור.

**מצב פלט:** קובע אם תשובות מופיעות כטקסט בלבד, כאודיו בלבד, כטקסט עם פלט קולי המופעל ידנית, או כטקסט עם פלט קולי אוטומטי. זיהוי השקט (500–3000 מ"ש, ברירת מחדל 1500 מ"ש) קובע כמה זמן ממתינים לאחר הדיבור לפני שההקלטה נשלחת אוטומטית.

**ניקוי תמלול:** אם האפשרות פעילה, טקסט ה-STT הגולמי עובר בנוסף דרך LLM המסיר מילות מילוי, מפרק תיקונים עצמיים ומחליק דקדוק ומספרים. את המודל המשמש לכך אתם בוחרים בעצמכם; באמצעות "מציאת המודל הטוב ביותר" Klacks בודק את כל המודלים המופעלים עם חלון הקשר מ-16,000 טוקנים ומעלה, ומציע את המועמדים הזולים והמהירים ביותר (זמן קצוב 30 שניות לכל מודל). ניתן לערוך את הנחיית הניקוי ולאפס אותה להגדרת המפעל.

**מילון:** עבור מונחים שזיהוי הדיבור מפרש לעיתים קרובות באופן שגוי (למשל שמות פרטיים או מונחים מקצועיים), אתם יוצרים רשומות עם המונח הנכון, קטגוריה, וריאציות הגייה ושיוך שפה אופציונלי — Klacksy מתקן אז מונחים אלה באופן ממוקד.

## טוב לדעת

- מצב הדפדפן אינו דורש מפתח API, אך תלוי בזיהוי הדיבור של הדפדפן הרלוונטי ואינו ניתן להשוואה באיכותו לשירותים בתשלום.
- מפתחות API מוצגים לאחר השמירה רק כ"מוגדר", לעולם לא בטקסט גלוי.
- בדיקת המודל כאן בודקת אך ורק את ההתאמה לניקוי תמלול (חלון הקשר ≥ 16,000 טוקנים) — עבור ההתאמה הכללית כמודל ראשי של Klacksy קיימת בדיקה נפרדת ומחמירה יותר (ראו בדיקת חיבור למודל).
