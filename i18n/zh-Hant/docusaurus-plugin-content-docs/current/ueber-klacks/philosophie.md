# Klacks 背後的理念

Klacks 是真正屬於您自己的人力排班系統 — 而不是屬於某個雲端服務商。有三項核心理念貫穿整個產品。

## 資料自主，而非依賴雲端

Klacks 運行在您自己的基礎架構之上 — 資料不必被迫離開您的機構。對於 Spitex 居家照護、醫院或保全服務這類敏感產業來說，這絕不只是錦上添花：把個人資料放在陌生的伺服器上、並受制於供應商，對這些機構而言往往根本不是選項。Klacks 顛覆了這種主從關係：資料存放在哪裡由您決定，而不是由軟體廠商決定。

## AI 自由選擇，而非受制於單一廠商

內建的 AI 助理 Klacksy 並不綁定任何單一 AI 供應商。無論是 OpenAI、Anthropic、Google、DeepSeek，還是自行架設的本地模型 — 成本、資料隱私與供應商的選擇，都由您決定，而不是由 Klacks 替您決定。因此，AI 輔助與完整的資料自主權並非互斥，而是可以並存。

## 開放原始碼，而非黑盒子

Klacks 以 MIT 授權條款開放原始碼 — 後端、前端與 Docker 映像檔皆公開可查閱。您可以檢視程式碼、將它 fork、進行自己的調整，或參與後續開發。這不是行銷口號，而是可以親自查證的事實：全部原始碼都公開放在 GitHub 上。

## 適合什麼樣的對象

Klacks 適合所有重視上述三項理念、需要排班與外勤調度的機構 — 尤其是 Spitex 居家照護機構、醫院、保全服務、清潔與物流業者。這些機構的共同點是：需要 24/7 全天候覆蓋、面對複雜的休息時間與資格要求，並且對資料保護有著一般可替換的雲端方案未必能滿足的高標準。

## 這在實務上代表什麼

- 您可以在 [Playground](https://klacks-software.ch:7643) 中免費試用 Klacks，不需要註冊，也不需要任何銷售洽談。
- 您可以透過 On-Premise 套件（Docker 映像檔、安裝程式、資料庫、HTTPS、自動更新）將它架設在自己的伺服器上運作。
- 您可以檢視原始碼並自行進行修改 — 即使您對它做了進一步開發，Klacks 依然屬於您。
