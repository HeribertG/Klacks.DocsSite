# 角色與權限

Klacks 針對登入帳號僅提供兩種可指派的權限層級：**主管（Supervisor）**與**管理員（Admin）**。並不存在第三種獨立的技術角色 — 下文中「排班負責人」之類的說法，指的是**日常工作中的一種職能**，而不是額外的權限層級。

## 兩種真正的角色

**主管**可以建立、編輯與刪除地址、群組、合約、缺勤與班次，也可以在排班表中核准某一天或某個群組，並可以再次撤回該核准。至於單純確認個人的工作時間，則不需要任何特殊角色 — 任何登入帳號都能做到。

**管理員**除了擁有主管的全部權限之外，還可以：存取所有設定（包含使用者管理本身）、結算並重新開啟期間，以及管理身分識別提供者、報表與日曆規則等受到特別保護的區域。

角色指派是透過使用者管理（齒輪圖示）中的下拉選單完成，並且立即生效，不需要另外點擊儲存。

## 「排班負責人」是一種職能，而非角色

在日常工作中，通常是由某個主管帳號來負責實際的排班工作 — 啟動自動排班、在排班網格中以拖放方式調整、將某天或某個群組送出核准。這並不是一個獨立的權限層級，而只是主管帳號本來就擁有的權限。

## 小提醒

- 目前登入的使用者也會顯示在使用者管理中，但無法在該處編輯自己、變更自己的權限或刪除自己。
- 永久結算或重新開啟期間，始終只保留給管理員 — 即使某位主管已經核准了相應的天數或群組。
- 登入帳號不一定要與員工的基礎資料卡片連結 — 使用者管理與人員管理是兩個各自獨立的區域。
