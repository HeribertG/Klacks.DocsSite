# 核准流程：從草稿到具約束力的排班表

在 Klacks 中，排班表會經過幾個明確定義的階段 — 讓團隊隨時都能清楚知道哪些內容是可以信賴的。

## 運作方式

排班表會依序經歷四個階段：

1. **草稿** — 工作階段：一切都還可以修改
2. **已確認** — 排班內容已經確定；之後的修改只能以受控方式進行
3. **已核准** — 負責單位已完成放行
4. **已結案** — 排班表已封存，具有約束力

每個階段都會相應地**鎖定編輯權限**：已核准的內容，不會有人再透過拖放不小心更動。誰能設定哪個階段，也有明確的規範：

- **確認**：任何已登入的使用者皆可執行。
- **核准**（以及撤回核准）：需要 **Supervisor（主管）** 或 **Admin（管理員）** 角色 — 且以日期與群組為單位分別授權。
- **結案**某個期間（以及日後重新開啟）：**僅限 Admin（管理員）**執行。

## 小提醒

- 鎖定機制在各處都同樣有效 — 就連[自動排班：數秒生成排班表](../planen/auto-planung.md)與 [Klacksy](../ki/klacksy-ki-assistent.md) 也都會遵守核准狀態。
- 核准後的臨時異動（例如同仁生病！）會以有意識、可追溯的更正方式處理，而不是悄悄重新排班。
- 若要對某個月份進行會計上的結算，另外還有[期間結算](../planen/periodenabschluss.md)可供使用。
