# 模組化排班：分而治之

與其艱難地應付一份龐大而混亂的總排班表，不如將它拆解成小而清楚的子排班表 — Klacks 會自動確保一切保持同步。

## 運作方式

1. **拆分** — 將龐大的排班表劃分為容易掌握的區塊：依團隊、部門或地點劃分。
2. **只需排班一次** — 在多個區塊工作的員工只需排班一次，就會自動正確顯示在每個相關區塊中。
3. **自動同步** — 一項變更會立即在各處同步更新。

其中的訣竅在於：各排班表保留的是**參照，而非複本**。一個班次只會存在一次 — 無論它出現在多少份子排班表中。這麼一來，重複排班就無所遁形：持續運作的衝突偵測機制，會在變更被正式儲存之前立即找出重疊之處。

## 小提醒

- 共用人員在其他區塊的班次，會在子排班表中顯示為鎖定條目 — 讓每位排班人員都能看到「自己的」人員何時被其他地方占用，但無法更動別人的排班表。
- 子排班表的可視範圍，可透過以群組為基礎的存取權限來控制。
- 模組化排班與[自動排班：數秒生成排班表](../planen/auto-planung.md)相輔相成：每個區塊都可以個別進行自動排班。
