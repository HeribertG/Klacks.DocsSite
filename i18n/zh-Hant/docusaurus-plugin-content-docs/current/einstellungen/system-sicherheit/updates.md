# 更新

在「設定 > 更新」中，您可以看到目前安裝的版本與最新可用的 Klacks 版本，控制自動更新行為，並在需要時手動觸發更新或回復。

## 運作方式

**狀態：** 卡片會顯示目前版本、是否有更新版本可用，以及目前執行中或最近一次執行的更新作業及其狀態。

**觸發更新或回復：** 點擊「立即安裝」可手動啟動可用的更新；「復原」則會撤銷最近一次成功的更新作業。只要已有作業正在執行，這兩個動作都會被鎖定。

**設定自動化：**
- **啟用自動更新** 用來開啟或關閉自動檢查與自動執行更新。
- **僅通知** 不會觸發自動更新，但會在有新版本可用時通知您。
- **通道** 可在 `Stable` 與 `Beta` 之間選擇更新來源。
- **檢查間隔**（小時）與**維護時段**（起訖時間）決定多久檢查一次，以及在哪個時間範圍內執行安裝。
- **保留備份數量** 決定在遷移前保留多少份備份，超出的舊備份會自動刪除。

**紀錄：** 一份表格列出最近幾次更新作業，包含類型、目標版本、狀態與請求時間。

## 小提醒

- 每次遷移之前，Klacks 都會自動建立備份 — 沒有備份就不會套用遷移。
- 若更新後的健康檢查失敗，Klacks 會自動回復到先前版本（還原備份 → 啟用舊版本 → 再次檢查）；只有在這次回復本身失敗時，才需要人工介入。
- 手動回復會針對性地還原最近一次成功作業所對應的備份，並啟用該作業的起始版本 — 而不是單純「退回上一版」。
- 更新與回復僅限管理員操作。
