在「設定 > 訊息通道」中，您可以串接外部聊天與簡訊通道，讓 Klacks（以及 Klacksy）能直接傳訊給員工與客戶 — 做為電子郵件之外的替代或補充方式。

## 運作方式

**共同原則：** 所有通道都透過同一份清單設定：內部名稱、顯示名稱、供應商類型，以及依通道而異的存取憑證（權杖、帳戶 ID、憑證等），另外每個項目都有一個開關與一個測試按鈕，可立即確認連線是否正常。訊息最終會送達誰，取決於在員工或客戶「地址 > 聯絡資訊」中登記的通道專屬識別碼（電話號碼、使用者名稱、使用者 ID 等）— 您自己不需要輸入任何原始 ID。

**Klacks 中訊息通道的用途：** 做為電子郵件以外的額外通知方式，例如用來傳送班表相關訊息，或 Klacksy 發出的訊息 — 實際使用哪一種，取決於啟用了哪些通道，以及每個人登記了哪些聯絡方式。

**11 種通道總覽：**

| 通道 | 傳送/接收 | 特別說明 |
|---|---|---|
| Slack | 傳送 + 接收 | 機器人還必須透過 `/invite` 明確邀請進入每個頻道。 |
| Telegram | 傳送 + 接收 | 收件人必須先自行透過 `/start` 啟用該機器人。 |
| Signal | 僅傳送 | 需要自行架設 signal-cli 橋接服務（Docker），並使用一組透過 QR code 綁定的獨立電話號碼 — 沒有官方雲端 API。 |
| Microsoft Teams | 僅傳送 | 每個頻道需要一組 Power Automate Webhook；不支援 1 對 1 訊息，多個 Teams 頻道須建立多筆設定。 |
| SMS | 僅傳送 | 電話號碼須為 E.164 格式；試用帳戶（例如 Twilio）只能傳送給事先驗證過的號碼，且目的地國家必須已開通。 |
| WeChat | 僅傳送 | 在中國以外註冊為官方帳號較為困難；只能觸及最近 48 小時內有過互動的關注者。 |
| KakaoTalk | 僅傳送 | 存取權杖約僅 6 小時有效，須手動更新；收件人必須是該應用程式的「好友」。 |
| Line | 傳送 + 接收 | 每月可傳送額度依方案而定；收件人 ID 要等對方傳來一則訊息後才能取得。 |
| Threema | 僅傳送 | 需要付費的 Threema Gateway 帳戶（儲值制）；因未使用端對端模式，故無法接收訊息。 |
| Viber | 傳送 + 接收 | 機器人只能傳訊給先前已訂閱它的使用者；Webhook 需要可從公開網路連線的 HTTPS 位址。 |
| Zalo | 僅傳送 | 存取權杖約 25 小時後過期，只能在互動時間窗內傳送訊息。 |

## 小提醒

- 每個通道都各自附有詳盡的手冊，逐步說明如何在對應供應商處完成設定 — 可直接在該供應商項目的設定頁面上查閱。
- 多個通道可以同時啟用；實際會使用哪一個，取決於該對象登記了哪些聯絡方式。
- 所有使用限時存取權杖的通道（例如 KakaoTalk、Zalo），都必須在 Klacks 之外向供應商更新該權杖，再回填到 Klacks 中，否則透過該通道的傳送會中斷。
