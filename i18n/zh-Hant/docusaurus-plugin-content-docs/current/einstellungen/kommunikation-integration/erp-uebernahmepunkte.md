在「設定 > ERP 匯入端點」中，您可以設定外部 ERP 系統的訂單如何以 XML 檔案的形式自動匯入 Klacks。

## 運作方式

**三種投遞方式：** ERP 系統可以將檔案放進匯入端點的檔案存放區（Klacks 會依排程自動抓取）；也可以透過 HTTP POST 直接推送到匯入端點（以存取權杖進行驗證）；此外，管理員也可以在這個頁面上以拖曳方式手動上傳 XML 檔案。

**匯入端點**是由 Klacks 自行管理的中央信箱，用來存放所有匯入檔案 — 分為三個區塊：收件（等待下一次執行）、已處理（歸檔）以及錯誤（含錯誤原因的無效檔案，可重新匯入）。排程（Cron 運算式，預設為每小時一次）以及一個用來暫停的啟用開關，共同控制自動抓取的行為。

**存取權杖**只有在 ERP 系統以推送方式投遞檔案時才需要。每個權杖都以 `klacks_erp_` 開頭，效期為 1 到 730 天（預設 365 天），並且**只會在建立時**以明碼顯示一次。這類權杖僅適用於匯入端點，不能用來存取其餘的 API，並且可以隨時撤銷。

**處理流程：** 訂單會依據來源系統 ID 與外部客戶參照號（若無則改用公司名稱／郵遞區號／街道）比對到既有客戶或建立新客戶，並先以草稿形式產生，需由排班人員審核並封存。已封存訂單若收到內容有變動的重新投遞，系統會自動結束舊訂單並建立一筆與其關聯的新版本；內容未變的重複投遞則不會觸發任何動作。

## 小提醒

- 個別有誤的訂單（例如缺少星期幾或參照號重複）會被拒絕，但不會阻擋同一檔案中其餘有效訂單的匯入；不過整個檔案仍會被歸入錯誤區塊，確保不會被遺漏。
- 已處理的檔案會被歸檔，而不會被刪除。
- 確切的 XML 格式（必填欄位、日期格式、星期幾邏輯）已直接記錄在設定頁面的手冊中，另外也提供可下載的範例檔案。
- 這項設定位於設定區域，因此僅限管理類角色使用。
