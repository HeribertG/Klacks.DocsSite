# 模型連線測試

並非每個語言模型都適合作為 Klacksy 的引擎 — 它必須能夠可靠地回傳結構化的函式呼叫，而不是以文字敘述作答。您可以在「設定 > Klacksy 模型檢測」中，讓 Klacks 針對所有已設定的模型自動執行這項測試。

## 運作方式

**測試內容：** 點擊「尋找最佳模型」後，系統會向每個已設定的模型（包括已停用的模型）送出同一個小測試任務：「新增一位名叫 Anna Müller 的員工。」模型必須以正確的結構化函式呼叫、搭配正確的姓名參數作答 — 而不是以文字回答。此外，上下文視窗必須至少達到 32,000 個 Token，才能容納 Klacksy 的系統提示詞與各項技能。每次測試的時間限制為 30 秒。

**結果：** 同時符合兩項條件的模型會被判定為「合適」並自動啟用；不合適的模型則會被停用。結果清單依序按適用性、函式呼叫能力、成本、延遲與上下文視窗排序。

**設定預設模型：** 透過「設為預設」，您可以直接在結果清單中選擇 Klacksy 日後預設使用的模型。若您不主動選擇，Klacks 會自動將最合適的模型設為預設。

## 小提醒

- 測試會檢查所有已建立的模型，而不只是已啟用的模型 — 先前被停用的模型，只要通過測試同樣會被啟用。
- 通過測試並不代表成本可控：昂貴但合適的模型同樣會被啟用。每 1000 個 Token 的成本可以在結果清單中查看。
- 這項測試專門針對支援行動能力的函式呼叫而設計，與語音設定中的模型檢測相互獨立 — 後者對於純粹的轉錄文字清理，允許使用較小的上下文視窗。
