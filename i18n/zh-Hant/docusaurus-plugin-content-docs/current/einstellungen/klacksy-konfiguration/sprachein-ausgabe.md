# 語音輸入與輸出

Klacksy 也可以透過語音操作 — 您可以在「設定 > Klacksy 語音」中，設定要用哪個服務將您的聲音轉成文字（語音轉文字，STT），以及 Klacksy 要用哪種聲音回答您（文字轉語音，TTS）。

## 運作方式

**語音辨識（STT）：** 可供選擇的引擎共有四種 — 免費的瀏覽器方案（Web Speech API）、Deepgram（即時串流）、Groq Whisper（速度快、成本低），以及 AssemblyAI（串流）。除了瀏覽器方案之外，其餘都需要您自行填入 API 金鑰，並可以直接測試連線。

**語音合成（TTS）：** 同樣有四家服務商可選 — Edge TTS（免費）、OpenAI TTS、ElevenLabs 與 Google Cloud TTS。依所選服務商不同，Klacks 會自動載入可用的語音；這裡同樣可以測試連線。

**輸出模式：** 決定回答要以純文字、純語音、可手動觸發語音播放的文字，還是自動語音播放的文字形式呈現。靜音偵測（500–3000 毫秒，預設 1500 毫秒）決定說話結束後要等待多久，錄音才會自動送出。

**轉錄文字清理：** 啟用此選項後，原始的 STT 文字會再經過一個 LLM 處理，去除贅字、消解自我修正，並潤飾文法與數字。用於此用途的模型由您自行選擇；透過「尋找最佳模型」，Klacks 會測試所有上下文視窗達 16,000 個 Token 以上的已啟用模型，並推薦其中最便宜、最快的候選（每個模型逾時時間為 30 秒）。清理提示詞可以編輯，也可以重設回原廠設定。

**詞典：** 對於語音辨識經常辨識錯誤的詞語（例如專有名稱或專業術語），您可以建立詞條，填入正確的詞語、類別、發音變體，以及選填的語言對應 — Klacksy 之後就會針對性地修正這些詞語。

## 小提醒

- 瀏覽器模式不需要 API 金鑰，但仰賴各瀏覽器本身的語音辨識能力，品質無法與付費服務相比。
- API 金鑰儲存後只會顯示為「已設定」，不會再以明文顯示。
- 此處的模型檢測只判斷是否適合用於轉錄文字清理（上下文視窗 ≥ 16,000 個 Token）— 若要判斷是否整體適合作為 Klacksy 的主要模型，另有專門且更嚴格的測試（參見「模型連線測試」）。
