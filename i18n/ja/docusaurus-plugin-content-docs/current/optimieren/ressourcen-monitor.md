リソースモニターは、選択した年におけるグループ(または全グループ)の人員稼働状況を一目で示します——1日ごとに1本のバーで、365日分の年間推移全体を表示します。

## 仕組み

この表示は、人員計画で定着している「キャパシティ対ワークロード・チャート」(プロジェクトマネジメント分野の文献では「リソースヒストグラム」として知られています)に基づいています。要素は以下の通りです。

- **緑色のバー — 勤務:** その日に計画されている勤務の数。各勤務は1人分のポジションとしてカウントされ、コンテナ勤務は1としてカウントされます。
- **灰色のバー — 欠勤:** 緑色のバーに積み重ねて表示される、不在の従業員の数。休暇や病欠の全日登録は1、半日登録は0.5としてカウントされます——週末も同様です。
- **ピンクの点線 — 希望する稼働可能人数:** 希望する週あたりの勤務日数が守られた場合に、平均してどれだけの従業員が稼働可能かを示します(標準:週5日勤務、2日休養)。これが計画の目標値です。
- **赤の破線 — 最大稼働可能人数:** 許容される最大連続勤務日数(標準:6日)における物理的・法的な上限値です。常にピンクの線と同じか、それ以上になります。
- **青色の線 — 従業員数:** その日に有効な契約を持つ全従業員数(ヘッドカウント)。

**読み方:** 緑色のバーの上端がピンクの線に近づくほど、人員は最適に活用されています。青色の線と積み重ねたバーの上端との差が、余力(リザーブ)です。

右上の矢印ボタンで年を切り替え、グループ選択で特定の拠点やチームに絞り込むことができます。

## 知っておくと便利なこと

- 経験則として、灰色の欠勤バーの上端は、勤務数と総人員数の差(ギャップ)の3分の1を超えないようにすべきです——それ以上になると運用上の余力が危険にさらされ、病欠や休暇のピーク時に穴を埋めるのが難しくなります。
- 表示される値は推定値・近似値であり、正確な集計ではありません——稼働可能状況の目安を示すものです。
- 勤務データは計画(プランニング)から取得されており、実績(実施済み)からではありません。そのため、将来の勤務も表示されます。
- 24時間365日体制の契約の場合、ピンクの線は週全体で平準化されます(1人・暦日あたり約0.71)。月〜金の契約の場合、平日は1.0、週末は0になります。
