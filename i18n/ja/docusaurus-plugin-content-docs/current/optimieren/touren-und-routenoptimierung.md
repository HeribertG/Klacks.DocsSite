# ツアー・ルート最適化

在宅ケアの巡回、警備巡回、清掃対象施設、配送といった移動を伴う業務のために、Klacksは*誰が*だけでなく、*どの順序で*行うかも計画します。

## 仕組み

各作業場所には住所が登録されているため、Klacksは経路を把握しています。**蟻コロニーアルゴリズム**(ルート問題向けに実績のある最適化手法)が、作業先の住所から最も合理的な順序を計算します。

1. **住所を登録** — 作業場所ごとに一度だけ登録します
2. **順序を最適化** — Klacksが最短で合理的な順序を決定します。回り道や空走が削減されます
3. **地図上に表示** — 完成したツアーは、車・自転車・徒歩のいずれかを選んでルートとして表示されます

ある業務に固定の時間枠が設定されている場合(例:午前中のみ)、それはツアー計画に反映されますが、厳守を保証するものではありません。

## 知っておくと便利なこと

- 距離と所要時間の計算のために、Klacksは外部サービスOpenRouteServiceを連携できます(その際、GPS座標は匿名化されて送信されます)——インスタンスの設定によります。
- ツアー最適化は、コンテナ勤務向けの独立したモジュールです。シフト計画の一般的な自動計画は、これとは独立してカバー率、公平性、ルール遵守を評価します。
- 交通手段はツアーごとに選択できます——都市部の自転車ツアーと地方の車ツアーを併用する場合に便利です。
- 誰かが欠勤した場合でも、Klacksは資格を持ち利用可能な人員を即座に表示します。
