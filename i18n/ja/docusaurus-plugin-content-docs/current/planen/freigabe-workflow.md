# 承認ワークフロー: 下書きから確定プランへ

Klacksでは、勤務表は明確に定義された段階を経て進んでいきます — そうすることで、チームが何を頼りにできるかが常に明確になります。

## 仕組み

プランは4つの段階を通過します。

1. **下書き** — 作業段階です。すべてがまだ変更可能です
2. **確認済み** — プランの内容は業務的に固まっており、変更は制御された形でのみ行われます
3. **承認済み** — 責任部署が承認を行いました
4. **完了** — プランは封印され、確定したものです

各段階は、それぞれに応じて**編集をロック**します。承認されたものは、誰かがドラッグ&ドロップでうっかり変更することはありません。どの段階を誰が設定できるかも、明確に定められています。

- **確認**は、ログインしている全員が行えます。
- **承認**(および承認の取り消し)には、**Supervisor**または**Admin**の役割が必要です — それぞれ日ごと・グループごとに設定されます。
- 期間の**完了**(および後からの再オープン)は、**Adminのみ**に限定されています。

## 知っておくと便利なこと

- ロックはあらゆる箇所に適用されます — [自動計画: 数秒でできるプラン](../planen/auto-planung.md)や[Klacksy](../ki/klacksy-ki-assistent.md)も、承認状態を尊重します。
- 承認後の急な変更(病欠など!)は、意図的で追跡可能な修正として扱われ、こっそりとした再計画ではありません。
- 月次の経理締めについては、別途[期間締め](../planen/periodenabschluss.md)があります。
