# モジュール式計画: 分割統治

見通しの悪い巨大なプランをやりくりする代わりに、それを小さく分かりやすいサブプランに分割します — 全体の整合性はKlacksが自動で保ちます。

## 仕組み

1. **分割する** — 大きな勤務表を、チームごと、部署ごと、あるいは拠点ごとに扱いやすい範囲に分けます。
2. **一度だけ計画する** — 複数の範囲で働く人は一度だけ計画すれば、どの範囲にも自動的に正しく表示されます。
3. **自動的に同期** — 変更は即座にあらゆる場所に反映されます。

その裏側の仕組みは、プランシートが**コピーではなく参照**を保持していることです。ある勤務は、いくつのサブプランに表示されようとも、実体として一つしか存在しません。そのため、二重登録がどこにも気づかれないまま残ることはありません。継続的に稼働している重複チェックが、確定保存される前に重なりを即座に検知します。

## 知っておくと便利なこと

- 共有されている人物の担当範囲外の勤務は、サブプラン内ではロックされたエントリーとして表示されます。こうすることで、計画担当者は「自分の」人材が他でどう拘束されているかを、他人のプランを変更することなく確認できます。
- サブプランの表示範囲は、グループベースのアクセス権限で制御します。
- モジュール式計画と[自動計画: 数秒でできるプラン](../planen/auto-planung.md)は互いに補完し合います。各範囲を個別に自動計画することができます。
