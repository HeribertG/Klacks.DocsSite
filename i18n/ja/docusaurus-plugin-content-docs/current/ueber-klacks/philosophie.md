# Klacksの背後にある哲学

Klacksは、クラウド事業者のものではなく、あなた自身のものである人員配置計画ソフトウェアです。製品全体を貫く決定が3つあります。

## クラウド依存ではなく、データ主権を

Klacksは、あなた自身のインフラ上で稼働します — データが必ずしも外部に出る必要はありません。訪問看護(Spitex)、病院、警備サービスといった機微なデータを扱う業種にとって、これは単なるあれば嬉しい機能ではありません。個人データが他社のサーバー上に置かれ、事業者に依存する状態は、こうした事業者にとって多くの場合そもそも選択肢になり得ません。Klacksはこの関係を逆転させます。データがどこに置かれるかを決めるのは、ソフトウェアメーカーではなく、あなた自身です。

## ベンダーロックインではなく、AI選択の自由を

内蔵AIアシスタントであるKlacksyは、特定のAI事業者に縛られていません。OpenAI、Anthropic、Google、DeepSeek、あるいはローカルで動作するモデル — コスト、データ保護、事業者について決めるのはKlacksではなく、あなたです。AIによる支援と完全なデータ主権は、互いに排他的なものではなく、本来一体のものなのです。

## ブラックボックスではなく、オープンソースを

Klacksは MIT ライセンスの下で公開されているオープンソースです — バックエンド、フロントエンド、Dockerイメージはすべて公開され、誰でも閲覧できます。コードを検証し、フォークし、独自の変更を加え、あるいは開発に貢献することができます。これは単なるマーケティング上の約束ではなく、検証可能な事実です。ソースコード全体がGitHub上に公開されています。

## 誰のためのものか

Klacksは、シフト制・出動制の計画を行い、これら3つの点を真剣に受け止める事業者、とりわけ訪問看護(Spitex)組織、病院、警備サービス、清掃・物流事業者を対象としています。共通するのは、24時間365日のカバー体制、複雑な休息時間・資格要件、そして代替可能なクラウド型ソリューションでは必ずしも満たせないデータ保護への要求です。

## 実際にはどういうことか

- [Playground](https://klacks-software.ch:7643)で、登録や商談なしに無料でKlacksを試すことができます。
- オンプレミス・パッケージ(Dockerイメージ、インストーラー、データベース、HTTPS、自動アップデート)を使って、自社のサーバー上で運用することができます。
- ソースコードを閲覧し、自ら変更を加えることができます — 開発を進めても、Klacksはあなたのものであり続けます。
