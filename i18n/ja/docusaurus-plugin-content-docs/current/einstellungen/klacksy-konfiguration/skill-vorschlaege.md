# Klacksy スキル提案

Klacksyは自らの誤りから学習します。チャット内でKlacksyが誤ったスキル（機能）を選択したことを修正すると、システムはその修正を記憶します。設定 > Klacksy スキル提案では、管理者はそこから提案されるスキル説明の改善内容を確認し、その採用について判断できます。

## 仕組み

**提案の出所:** チャット内でKlacksyがリクエストに対して誤ったスキル（機能）を選択し、ユーザーがそれを修正すると、そのチャットのシーケンスは「修正済み」としてマークされます。内部の最適化エンジンはこのような修正済みシーケンスを分析し、そこから該当スキルの説明文に対する具体的なテキスト変更を導き出します — 目的は、次回Klacksyが該当するスキルを正しく割り当てられるようにすることです。

**提案を生成:** 「提案を生成」ボタンは、直近30件の修正済みチャットシーケンスの分析を開始し、パターンが見つかった場合はそこから新しい提案を生成します。

**提案カードに表示される内容:** 該当するスキルと変更されたフィールド、変更前後の比較、その変更が妥当である理由の説明、そして展開表示できる形で、提案のきっかけとなった具体的なユーザーリクエストです。

**判断:** 未処理の提案はそれぞれ個別に**採用**（スキルの説明文が即座に変更されます）または**却下**（何も変更されずに提案が破棄されます）されます。自動的な採用は行われません — 常に人間が判断に関与します。

## 知っておくと良いこと

- チャット内で修正が行われなければ提案は発生しません — システムには単なる利用実績ではなく、実際の誤り修正が学習の基盤として必要です。
- この機能は、Klacksyが正しく選択できるようにスキルの説明文のみを変更します。スキルが実際に何を行うかを変更するものではありません。
- この機能には管理者権限が必要です。
