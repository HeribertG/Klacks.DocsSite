# モデル接続テスト

すべての言語モデルがKlacksyに適しているわけではありません — 散文で答えるのではなく、構造化された関数呼び出しを確実に返す必要があります。設定 > Klacksyモデルチェックで、Klacksは設定済みのすべてのモデルに対してこれを自動的にテストします。

## 仕組み

**テスト内容:** 「最適なモデルを探す」をクリックすると、設定済みのすべてのモデル(無効化されているものも含む)に同じ小さなテスト課題が送られます: 「Anna Müllerという名前の新しい従業員を登録してください。」モデルはこれに対し、テキストによる回答ではなく、適切な構造化された関数呼び出しと正しい名前パラメータで応答する必要があります。さらに、Klacksyのシステムプロンプトと機能が収まるよう、コンテキストウィンドウは少なくとも32,000トークンを備えている必要があります。各テストは30秒のタイムリミットで実行されます。

**結果:** 両方の基準を満たすモデルは「適合」と判定され、自動的に有効化されます。適合しないモデルは無効化されます。結果リストは、適合性、次に関数呼び出し能力、コスト、レイテンシ、コンテキストウィンドウの順にソートされます。

**標準モデルの設定:** 「標準として設定」により、結果リストから直接、Klacksyが今後標準で使用するモデルを選択できます。自分で選択しない場合、Klacksは最も適合したモデルを自動的に標準として設定します。

## 知っておくと良いこと

- このテストは、有効化されているものだけでなく、登録されているすべてのモデルをチェックします — これまで無効化されていたモデルも、テストに合格すれば有効化されることがあります。
- テストに合格したからといってコストが保証されるわけではありません: 高価だが適合するモデルも同様に有効化されます。1000トークンあたりのコストは結果リストに表示されます。
- このテストは行動能力のための関数呼び出しに特化して設計されており、言語設定にあるモデルチェック(純粋な文字起こしの整形にはより低いコンテキストウィンドウで十分とするもの)とは独立しています。
