# ユーザー管理

「設定 > ユーザー管理」では、Klacksへのログインに使う**ログインアカウント**(システムユーザー)を管理します。これは従業員管理や住所管理とは異なるものです。ログインアカウントは人物マスタデータのカードと紐づけることができますが、必須ではありません。

## 仕組み

**ユーザーの追加:** 「+ ユーザーを追加」ダイアログでは、名と姓(それぞれ最低2文字)からユーザー名の候補が自動生成されます。メールアドレスは必須で、@マークを含んでいる必要があります。すべての項目が有効に入力されて初めて保存できます。作成時にはパスワードが自動生成されます ― 本人が自分でパスワードを設定できるようにするには、その後、該当行の「パスワードをリセット」を使用してください。

**ユーザーの編集:** 名前の行をダブルクリックすると編集ダイアログが開きます。そこでユーザー名を後から変更できます ― 新規作成時にはあくまで候補にすぎません。

**パスワードのリセット:** ユーザー行の「パスワードをリセット」リンクは、登録済みのアドレス宛てにメールを送信し、本人がそこから新しいパスワードを設定できるようにします。

**ユーザーの削除:** ゴミ箱ボタンで、削除ダイアログでの確認後にログインアカウントを完全に削除します。

**権限の割り当て:** 行内のドロップダウンから**Supervisor**または**Admin**を割り当てます。変更は別途「保存」をクリックすることなく即座に反映されます。この2つのロールが具体的に何を許可されているかは、「はじめに」の「ロールと権限」で説明されています ― ここではアカウント自体の管理のみを扱います。

## 知っておくと良いこと

- 現在ログイン中のユーザーもリストに表示されますが、そこで自分自身を編集したり、自分の権限を変更したり、削除したりすることはできません。
- ログインアカウントは人物・従業員管理とは独立しています ― 例えば、外部IT会社用のアカウントを、そのために従業員カードを作成することなく作成できます。
- 管理者が直接パスワードを設定することはできません。新規アカウントには自動生成されたパスワードが割り当てられ、その後の変更はすべてメールの「パスワードをリセット」リンク経由で行われます。
