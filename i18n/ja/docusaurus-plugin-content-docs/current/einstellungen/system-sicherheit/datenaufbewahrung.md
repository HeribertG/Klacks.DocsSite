# データ保持

「設定 > データ保持」では、削除されたデータ ― 従業員、住所、グループ、サービス、勤務時間、休暇、休憩などのレコード ― が、完全に削除されるまでシステム内にどのくらいの期間保持されるかを設定します。

## 仕組み

**2段階削除:** Klacksでレコードを削除すると、最初はソフトデリートのマークが付けられるだけです(削除済みとしてマークされますが、物理的にはまだ存在します)。ここで設定した期間が経過して初めて、毎日実行されるバックグラウンドサービスが、削除済みとマークされた行を**物理的かつ完全に**データベースから削除します ― 単一の領域だけでなく、影響を受けるすべてのテーブルにわたって処理されます。

**保持期間の設定:** 入力欄には30日から36,500日(約1ヶ月から100年)の値を指定できます。Klacksは併せて、この期間を年/月単位でも表示します。デフォルト値は3,650日(10年)です。

## 知っておくと良いこと

- 期間を短縮すると、すでに削除済みの既存レコードにも影響します。新しい期間よりも古いものはすべて、バックグラウンドサービスの次回実行時に削除されます ― 短縮する前に、それが意図した動作かどうかを確認してください。
- 物理削除は最終的なものです ― その後、サポートを含め、いかなる方法でも復元はできません。
- この設定は、プライバシーポリシーに記載されている削除期間の技術的な基盤です。全てのソフトデリート対応データ型にグローバルに適用され、モジュールごとに個別設定することはできません。
