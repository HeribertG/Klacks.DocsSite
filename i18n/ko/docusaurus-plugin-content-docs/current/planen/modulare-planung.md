# 모듈형 계획: 나누어 다스리기

한눈에 파악하기 어려운 거대한 계획표를 다루는 대신, 이를 작고 명확한 부분 계획으로 나눌 수 있습니다 — 그리고 Klacks가 자동으로 모든 것을 하나로 유지합니다.

## 작동 방식

1. **분할** — 큰 근무 계획표를 팀, 부서 또는 지점별로 관리하기 쉬운 영역으로 나눕니다.
2. **한 번만 배치** — 여러 영역에서 근무하는 사람은 한 번만 배치되며, 모든 곳에서 자동으로 올바르게 표시됩니다.
3. **자동 동기화** — 변경 사항은 즉시 모든 곳에 반영됩니다.

이 원리의 핵심은 계획표가 **복사본이 아니라 참조**를 유지한다는 점입니다. 하나의 근무는 몇 개의 부분 계획에서 보이든 상관없이 단 한 번만 존재합니다. 이 덕분에 중복 배정이 어디에서도 눈에 띄지 않게 발생할 수 없습니다: 지속적인 충돌 검사가 최종 저장 전에 즉시 겹치는 부분을 감지합니다.

## 알아두면 좋은 점

- 여러 영역에 공유된 사람의 다른 영역 근무는 부분 계획에서 잠긴 항목으로 표시됩니다 — 이를 통해 각 계획 담당자는 "자신의" 인력이 다른 곳에서 이미 배정되어 있는 시점을 확인할 수 있으며, 다른 사람의 계획을 변경할 수는 없습니다.
- 부분 계획의 표시 범위는 그룹 기반 접근 권한으로 제어합니다.
- 모듈형 계획과 [자동 계획: 몇 초 만에 완성되는 계획](../planen/auto-planung.md)은 서로 보완됩니다: 각 영역을 개별적으로 자동 계획할 수 있습니다.
