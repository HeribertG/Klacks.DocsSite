# 승인 워크플로: 초안에서 확정 계획까지

Klacks에서 근무 계획은 명확히 정의된 단계를 거칩니다 — 팀이 언제든 무엇을 신뢰할 수 있는지 명확히 알 수 있도록 하기 위해서입니다.

## 작동 방식

계획은 네 단계를 거칩니다.

1. **초안** — 작업 단계: 모든 것이 아직 변경될 수 있습니다.
2. **확정됨** — 계획이 실무적으로 확정되었으며, 변경은 통제된 방식으로만 가능합니다.
3. **승인됨** — 책임 부서가 승인을 완료했습니다.
4. **완료됨** — 계획이 봉인되어 구속력을 갖습니다.

각 단계는 그에 맞게 **편집을 잠급니다**: 승인된 내용은 누구도 드래그 앤 드롭으로 실수로 변경할 수 없습니다. 어떤 역할이 어떤 단계를 설정할 수 있는지는 명확히 규정되어 있습니다.

- **확정**은 로그인한 모든 사용자가 할 수 있습니다.
- **승인**(및 승인 철회)은 **Supervisor** 또는 **Admin** 역할이 필요하며, 각각 일자와 그룹 단위로 적용됩니다.
- 기간의 **완료 처리**(및 이후의 재개방)는 **Admin만** 수행할 수 있습니다.

## 알아두면 좋은 점

- 이 잠금은 모든 곳에 적용됩니다 — [자동 계획: 몇 초 만에 완성되는 계획](../planen/auto-planung.md)과 [Klacksy](../ki/klacksy-ki-assistent.md)도 승인 상태를 존중합니다.
- 승인 이후의 단기적인 변경(병가 등!)은 조용히 재계획하는 것이 아니라, 의식적이고 추적 가능한 수정으로 처리됩니다.
- 한 달의 회계 마감을 위해서는 별도로 [기간 마감](../planen/periodenabschluss.md) 기능이 있습니다.
