# 업데이트

설정 > 업데이트에서는 현재 설치된 버전과 최신으로 사용 가능한 Klacks 버전을 확인하고, 자동 업데이트 동작을 제어하며, 필요할 경우 수동으로 업데이트나 롤백을 실행할 수 있습니다.

## 작동 방식

**상태:** 카드에는 현재 버전, 최신 버전 사용 가능 여부, 그리고 현재 진행 중이거나 마지막으로 실행된 업데이트 작업과 그 상태가 표시됩니다.

**업데이트 실행 또는 롤백:** "지금 설치"를 클릭하면 사용 가능한 업데이트를 수동으로 시작합니다. "되돌리기"는 마지막으로 성공한 업데이트 작업을 취소합니다. 이미 다른 작업이 진행 중인 동안에는 두 작업 모두 잠겨 있습니다.

**자동화 구성:**
- **자동 업데이트 활성화**는 자동 업데이트 확인 및 실행 여부를 켜고 끕니다.
- **알림만 받기**는 자동 업데이트를 실행하지 않지만, 새 버전을 사용할 수 있을 때 알려줍니다.
- **채널**은 업데이트 소스로 `Stable`과 `Beta` 중에서 선택합니다.
- **확인 간격**(시간)과 **유지보수 시간대**(시작/종료 시각)는 얼마나 자주, 어떤 시간대에 확인 및 설치를 수행할지 지정합니다.
- **보관할 백업 수**는 마이그레이션 전에 몇 개의 백업을 보관할지, 그리고 오래된 백업을 언제 자동으로 삭제할지 결정합니다.

**이력:** 표에는 최근 업데이트 작업이 종류, 대상 버전, 상태, 요청 시각과 함께 나열됩니다.

## 알아두면 좋은 점

- 모든 마이그레이션 전에 Klacks는 자동으로 백업을 생성합니다 — 백업이 없으면 마이그레이션이 적용되지 않습니다.
- 업데이트 후 상태 점검(health check)이 실패하면 Klacks는 자동으로 이전 버전으로 롤백합니다(백업 복원 → 이전 버전 활성화 → 재점검). 이 롤백 자체가 실패하는 경우에만 수동 개입이 필요합니다.
- 수동 롤백은 마지막으로 성공한 작업에 해당하는 백업을 정확히 복원하고 그 원본 버전을 활성화합니다 — 단순히 "한 버전 이전으로" 되돌리는 것이 아닙니다.
- 업데이트와 롤백은 관리자 전용 작업입니다.
