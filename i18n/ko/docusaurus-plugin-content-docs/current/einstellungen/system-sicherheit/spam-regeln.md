# 스팸 규칙

설정 > 스팸 규칙에서는 받은편지함 모듈로 들어오는 이메일을 자동으로 스팸으로 분류하는 필터 규칙을 정의합니다 — 이 규칙은 새 이메일을 지속적으로 가져오는 과정(IMAP 폴링)뿐 아니라, 이미 수신된 이메일을 나중에 다시 평가할 때도 적용됩니다.

## 작동 방식

**규칙 생성:** 규칙 유형(발신자 포함, 발신자 도메인, 제목 포함, 내용 포함 중 하나)을 선택하고, 확인할 패턴을 입력한 다음 추가를 클릭합니다.

**규칙 관리:** 각 규칙은 체크박스를 통해 삭제하지 않고도 활성화하거나 비활성화할 수 있으며, 휴지통 버튼을 통해 영구적으로 삭제할 수도 있습니다.

## 알아두면 좋은 점

- 비활성화된 규칙은 저장된 상태로 유지되지만 새로운 이메일이나 재평가되는 이메일에는 더 이상 적용되지 않습니다 — 규칙을 삭제하는 대신 임시로 중단해 두고 싶을 때 유용합니다.
- Klacks는 ERP 주문 가져오기에도 일반 이메일 받은편지함 모듈과 동일한 받은편지함을 사용합니다(단일 사서함 아키텍처). 따라서 너무 광범위하게 설정된 규칙(예: 발신자 도메인 전체)은 이론적으로 수신되는 ERP 주문 이메일까지 스팸으로 분류할 수 있습니다 — 새 규칙을 활성화하기 전에 이 점을 신중하게 확인하세요.
- 원치 않는 발신 조직 전체를 차단하려면, 개별 주소마다 발신자-포함 규칙을 만드는 대신 발신자 도메인 규칙을 사용하세요.
