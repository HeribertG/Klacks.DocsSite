# 사용자 관리

설정 > 사용자 관리에서는 사람들이 Klacks에 로그인하는 데 사용하는 **로그인 계정**(시스템 사용자)을 관리합니다 — 이는 직원 관리나 주소 관리와는 다릅니다. 로그인 계정은 인물 기준 데이터 카드와 연결될 수도 있지만, 반드시 연결되어야 하는 것은 아니기 때문입니다.

## 작동 방식

**사용자 추가:** "+ 사용자 추가" 대화상자는 이름과 성(각각 최소 2자)을 바탕으로 사용자 이름을 자동으로 제안합니다. 이메일 주소는 필수이며 @ 기호를 포함해야 합니다. 모든 필드가 올바르게 입력되어야만 저장할 수 있습니다. 생성 시 비밀번호가 자동으로 생성됩니다 — 해당 사용자가 직접 비밀번호를 설정할 수 있도록 하려면, 이후 해당 행에서 "비밀번호 재설정"을 사용하세요.

**사용자 편집:** 이름 행을 더블클릭하면 편집 대화상자가 열립니다. 여기서 사용자 이름을 나중에 변경할 수 있습니다 — 신규 생성 시에는 어디까지나 제안일 뿐입니다.

**비밀번호 재설정:** 사용자 행에 있는 "비밀번호 재설정" 링크는 등록된 이메일 주소로 메일을 발송하며, 해당 사용자는 이 메일을 통해 직접 새 비밀번호를 설정합니다.

**사용자 삭제:** 휴지통 버튼은 삭제 대화상자에서 확인한 후 로그인 계정을 완전히 제거합니다.

**권한 부여:** 행에 있는 드롭다운을 통해 **Supervisor** 또는 **Admin**을 지정합니다. 변경 사항은 별도의 저장 클릭 없이 즉시 적용됩니다. 이 두 역할이 구체적으로 무엇을 할 수 있는지는 "시작하기"의 "역할과 권한"에서 설명합니다 — 여기서는 계정 자체의 관리만 다룹니다.

## 알아두면 좋은 점

- 현재 로그인한 사용자도 목록에 표시되지만, 본인 스스로는 편집하거나 권한을 변경하거나 삭제할 수 없습니다.
- 로그인 계정은 인물/직원 관리와 무관하게 독립적으로 존재합니다 — 예를 들어 외부 IT 업체를 위한 계정을 직원 카드 없이 생성할 수 있습니다.
- 관리자가 직접 비밀번호를 설정하는 기능은 없습니다: 새 계정은 자동으로 생성된 비밀번호를 받으며, 이후의 모든 변경은 이메일 링크 "비밀번호 재설정"을 통해 이루어집니다.
