# 데이터 보존

설정 > 데이터 보존에서는 삭제된 레코드(직원, 주소, 그룹, 근무, 근무 시간, 결근, 휴식 시간 등)가 완전히 제거되기 전까지 시스템에 얼마나 오래 보관될지를 지정합니다.

## 작동 방식

**2단계 삭제:** Klacks에서 레코드를 삭제하면 처음에는 소프트 삭제(Soft-Delete) 표시만 됩니다(삭제로 표시되지만 물리적으로는 아직 존재). 여기서 설정한 기간이 지난 후에야 매일 실행되는 백그라운드 서비스가 삭제 표시된 행을 데이터베이스에서 **물리적으로 되돌릴 수 없이** 제거합니다 — 특정 영역 하나가 아니라 영향을 받는 모든 테이블에 걸쳐 적용됩니다.

**보존 기간 설정:** 입력란은 30일에서 36,500일(약 1개월에서 100년) 사이의 값을 허용합니다. Klacks는 이와 함께 해당 기간을 연/월 단위로도 표시합니다. 기본값은 3,650일(10년)입니다.

## 알아두면 좋은 점

- 기간을 단축하면 이미 오래전에 삭제된 기존 레코드에도 영향을 미칩니다: 새 기간보다 오래된 모든 항목은 백그라운드 서비스가 다음번 실행될 때 제거됩니다 — 기간을 단축하기 전에 이것이 의도한 결과인지 확인하세요.
- 물리적 삭제는 최종적입니다 — 이후에는 지원팀을 통해서도 복구할 수 없습니다.
- 이 설정은 개인정보 처리방침에 기술된 삭제 기간의 기술적 기반입니다. 모든 소프트 삭제 가능 데이터 유형에 전역적으로 적용되며, 모듈별로 별도 설정할 수는 없습니다.
