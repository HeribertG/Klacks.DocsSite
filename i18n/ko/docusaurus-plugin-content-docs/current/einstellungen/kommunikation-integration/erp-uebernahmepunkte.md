설정 > ERP 수집 지점 메뉴에서 외부 ERP 시스템의 주문을 XML 파일 형태로 Klacks에 자동으로 수집하는 방법을 설정합니다.

## 작동 방식

**세 가지 전달 방식:** ERP 시스템은 드롭 포인트의 파일 저장소에 파일을 제공할 수 있으며(Klacks가 일정에 따라 자동으로 가져옵니다), HTTP POST를 통해 직접 임포트 엔드포인트로 푸시할 수 있고(접근 토큰으로 인증됨), 관리자가 이 페이지에서 XML 파일을 드래그 앤 드롭으로 직접 수동 업로드할 수도 있습니다.

**드롭 포인트**는 모든 임포트 파일을 위한 Klacks가 직접 관리하는 중앙 사서함으로, 세 가지 영역을 보여줍니다: 수신함(다음 실행을 대기 중), 처리됨(보관소), 오류(오류 사유가 포함된 유효하지 않은 파일, 재임포트 가능). 일정(Cron 표현식, 기본값 매시간)과 일시 중지를 위한 활성화 스위치가 자동 조회를 제어합니다.

**접근 토큰**은 ERP 시스템이 파일을 푸시 방식으로 전달할 때만 필요합니다. 각 토큰은 `klacks_erp_`로 시작하며, 1~730일(기본값 365일) 동안 유효하고, 생성 시 **단 한 번만** 평문으로 표시됩니다. 이러한 토큰은 임포트 엔드포인트에만 적용되며 나머지 API 접근에는 사용되지 않고, 언제든 폐기할 수 있습니다.

**처리:** 주문은 소스 시스템 ID와 외부 고객 참조번호(또는 대체로 회사명/우편번호/도로명)를 기준으로 기존 고객 또는 새로 생성된 고객에게 연결되며, 우선 초안 상태로 생성되어 플래너가 검토하고 확정해야 합니다. 이미 확정된 주문에 대한 변경된 재배송은 기존 주문을 자동으로 종료하고 새로운 연결된 버전을 생성합니다. 변경되지 않은 반복 배송은 아무런 작업도 트리거하지 않습니다.

## 알아두면 좋은 점

- 개별 주문에 오류가 있는 경우(예: 요일 누락 또는 참조번호 중복), 해당 주문은 거부되지만 같은 파일 내 다른 유효한 주문의 처리를 막지는 않습니다. 다만 해당 파일 전체는 누락되지 않도록 오류 영역에 남습니다.
- 처리된 파일은 삭제되지 않고 보관됩니다.
- 정확한 XML 형식(필수 필드, 날짜 형식, 요일 로직)은 설정 페이지에 직접 포함된 매뉴얼과 다운로드 가능한 예제 파일에 문서화되어 있습니다.
- 이 설정은 설정 영역에 있으므로 관리자 역할에만 허용됩니다.
