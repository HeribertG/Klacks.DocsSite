설정 > IMAP 설정 메뉴에서 Klacks가 백그라운드에서 지속적으로 조회하여 "받은 편지함" 영역에 표시할 메일함을 등록합니다 — 이를 통해 수신 서신을 직원 및 고객에게 자동으로 연결하여 확인할 수 있습니다.

## 작동 방식

**연결 정보:** 서버, 포트(기본값 993), SSL 암호화, 사용자 이름, 비밀번호, 조회할 폴더(기본값 `INBOX`), 조회 간격(초 단위, 기본값 300).

**조회한 메일의 처리 방식:** Klacks는 IMAP을 통해 주기적으로 메일함을 조회하고, 새 메시지를 스팸 필터로 검사하며(스팸은 Klacks와 실제 메일함 모두에서 정크 폴더로 이동됩니다), 나머지 메시지는 발신자 주소가 연락처 데이터로 등록되어 있는 경우 해당 고객 또는 직원 레코드에 자동으로 연결합니다. 이렇게 해서 "받은 편지함"에서 인물별로 필터링 가능한 서신 이력이 만들어집니다. "읽음으로 표시", 이동, 삭제와 같은 변경 사항은 실제 메일함에도 반영됩니다.

**연결 테스트:** "IMAP 연결 테스트" 버튼은 서버에 실제로 연결하여 로그인하고, 설정된 폴더를 읽기 전용으로 열어 발견된 메시지 수를 표시합니다. 폴더가 존재하지 않으면 이를 명시적으로 알립니다.

## 알아두면 좋은 점

- 이 메일함은 결근 신고를 위한 채널이 **아니며**, ERP 주문 수집의 일부도 **아닙니다**(이를 위한 별도의 수집 지점이 있으며, 별도 페이지 참고) — 오로지 일반 "받은 편지함"에만 데이터를 공급합니다.
- 이 설정은 전체 설치본에 전역으로 적용됩니다 — 사용자별 별도의 IMAP 계정은 없습니다.
- 설치본의 암호화에 사용되는 키가 초기화되면 Klacks는 저장된 비밀번호를 더 이상 복호화할 수 없으며, 비밀번호를 다시 저장할 때까지 조회를 일시 중지합니다.
- DeepL이 설정되어 있으면 받은 편지함의 메시지를 바로 번역할 수 있습니다(별도 페이지 참고).
