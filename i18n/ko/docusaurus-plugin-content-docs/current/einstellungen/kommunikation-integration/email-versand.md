설정 > 이메일 설정 메뉴에서 Klacks가 발신 이메일을 전송하는 데 사용하는 SMTP 서버를 등록합니다 — 예를 들어 알림, 비밀번호 재설정 메일 또는 Klacksy가 발송하는 메시지 등에 사용됩니다.

## 작동 방식

**연결 정보:** 서버, 포트, 타임아웃, SSL/TLS 모드, 인증 방식(없음, LOGIN, NTLM, GSSAPI, WDIGEST), 사용자 이름과 비밀번호를 입력합니다. 추가로 읽음 확인/전달 알림과 회신 주소를 설정할 수 있습니다. SSL/TLS 모드는 포트에 따라서도 자동으로 결정됩니다. 포트 465는 직접 SSL을 사용하고, 포트 587(또는 SSL 활성화 시)은 StartTLS를 사용합니다.

**저장:** 별도의 저장 버튼은 없습니다 — 각 필드는 변경 후 잠시 지연되었다가 자동으로 저장됩니다.

**연결 테스트:** "테스트 이메일 보내기" 버튼은 실제 SMTP 연결을 맺고 인증한 뒤 등록된 사용자 주소로 실제 테스트 이메일을 발송합니다. 이는 단순한 연결 확인이 아니라 실제 메일 발송이며, 접속 정보를 즉시 검증하는 데 유용합니다. 잘 알려진 제공업체(Outlook, Gmail, Yahoo, GMX 등)의 경우 Klacks는 "없음"이 아닌 인증 방식을 반드시 요구합니다.

## 알아두면 좋은 점

- 이 설정은 전체 설치본에 전역으로 적용됩니다 — 사용자별 별도의 SMTP 계정은 없습니다.
- 관리자 권한을 가진 사람만 이 페이지를 설정할 수 있습니다.
- 비밀번호는 암호화되어 저장되며, 입력 양식에서는 눈 모양 아이콘으로 표시/숨김이 가능하지만 평문으로 표시되는 일은 없습니다.
- 연결 테스트가 실패하면 Klacks는 일반적인 오류 메시지 대신 구체적인 원인(예: 인증 오류, SSL 핸드셰이크 오류, 시간 초과)을 알립니다.
