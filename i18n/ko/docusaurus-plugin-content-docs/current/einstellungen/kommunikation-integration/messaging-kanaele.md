설정 > 메시징 제공자 메뉴에서 Klacks(및 Klacksy)가 이메일 대신 또는 이메일과 함께 직원 및 고객에게 직접 메시지를 보낼 수 있는 외부 채팅 및 SMS 채널을 연동합니다.

## 작동 방식

**공통 원칙:** 모든 채널은 동일한 목록을 통해 설정됩니다: 내부 이름, 표시 이름, 제공자 유형, 채널마다 다른 접속 정보(토큰, 계정 ID, 인증서 등), 그리고 각 항목마다 켜기/끄기 스위치와 연결을 즉시 확인할 수 있는 테스트 버튼이 있습니다. 메시지가 누구에게 전달되는지는 직원 또는 고객의 주소 > 연락처 데이터에 등록된 채널별 식별자(전화번호, 사용자 이름, 사용자 ID 등)를 통해 결정됩니다 — 직접 원시 ID를 입력할 필요는 없습니다.

**Klacks에서 메시징이 사용되는 용도:** 이메일 외에 추가적인 알림 경로로 사용됩니다. 예를 들어 근무표에서의 알림이나 Klacksy가 보내는 메시지 등이며, 어떤 채널이 활성화되어 있고 각 사람에게 등록되어 있는지에 따라 달라집니다.

**11개 채널 개요:**

| 채널 | 송신/수신 | 특이사항 |
|---|---|---|
| Slack | 송신 + 수신 | 봇을 각 채널에 `/invite` 명령으로 별도로 초대해야 합니다. |
| Telegram | 송신 + 수신 | 수신자가 사전에 `/start` 명령으로 봇을 직접 활성화해야 합니다. |
| Signal | 송신 전용 | QR 코드로 연결된 자체 전화번호를 가진 자체 운영 signal-cli 브리지(Docker)가 필요합니다 — 공식 클라우드 API는 없습니다. |
| Microsoft Teams | 송신 전용 | 채널당 하나의 Power Automate 웹훅이 필요하며, 1:1 메시지는 불가능합니다. 여러 Teams 채널을 사용하려면 여러 항목이 필요합니다. |
| SMS | 송신 전용 | 전화번호는 E.164 형식이어야 합니다. 체험(Trial) 계정(예: Twilio)의 경우 사전에 인증된 번호로만 발송 가능하며, 대상 국가가 활성화되어 있어야 합니다. |
| WeChat | 송신 전용 | 중국 외 지역에서 공식 계정으로 인증받기가 까다롭습니다. 최근 48시간 이내에 상호작용한 팔로워에게만 도달합니다. |
| KakaoTalk | 송신 전용 | 접근 토큰은 약 6시간만 유효하며 수동으로 갱신해야 합니다. 수신자는 앱의 "친구"여야 합니다. |
| Line | 송신 + 수신 | 요금제에 따라 월간 발송 한도가 제한됩니다. 수신자 ID는 수신 메시지가 들어와야 비로소 알 수 있습니다. |
| Threema | 송신 전용 | 유료 Threema 게이트웨이 계정(잔액 모델)이 필요합니다. 종단간 암호화 모드를 사용하지 않으므로 수신은 불가능합니다. |
| Viber | 송신 + 수신 | 봇은 사전에 자신을 구독한 사람에게만 메시지를 보낼 수 있습니다. 웹훅에는 공개적으로 접근 가능한 HTTPS가 필요합니다. |
| Zalo | 송신 전용 | 접근 토큰은 약 25시간 후 만료되며, 발송은 상호작용 시간 창 내에서만 가능합니다. |

## 알아두면 좋은 점

- 각 채널마다 해당 제공자에서의 설정 방법을 단계별로 안내하는 전용 상세 매뉴얼이 있습니다 — 각 제공자 항목의 설정 페이지에서 바로 확인할 수 있습니다.
- 여러 채널을 동시에 활성화할 수 있습니다. 실제로 어떤 채널이 사용되는지는 해당 인물에게 어떤 연락처 데이터가 등록되어 있는지에 따라 달라집니다.
- 유효 기간이 있는 접근 토큰을 사용하는 모든 채널(예: KakaoTalk, Zalo)의 경우, Klacks 외부의 제공자 측에서 토큰을 갱신하여 Klacks에 다시 입력해야 합니다. 그렇지 않으면 해당 채널을 통한 발송이 중단됩니다.
