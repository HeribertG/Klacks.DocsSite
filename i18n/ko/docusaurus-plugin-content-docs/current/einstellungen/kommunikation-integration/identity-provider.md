설정 > Identity Provider 메뉴에서 Klacks를 외부 디렉터리 서비스 — LDAP/Active Directory 또는 OAuth2/OpenID Connect — 와 연결하여 직원을 자동으로 가져오거나 로그인을 이를 통해 처리할 수 있습니다.

## 작동 방식

**LDAP/Active Directory:** 호스트, 포트(389/636), SSL, Base DN, Bind DN, Bind 비밀번호, 사용자 필터를 등록합니다. "클라이언트 임포트에 사용" 스위치를 켜면 자동 임포트가 활성화됩니다. 이름, 성, 직함, 회사, 주소가 디렉터리 속성에서 가져와집니다 — OpenLDAP과 Active Directory에서는 일부 속성명이 서로 다릅니다(예: `uid` 대 `sAMAccountName`, `street` 대 `streetAddress`). 이러한 필드 매핑은 백엔드에 고정되어 있으며 화면에서는 편집할 수 없습니다.

**OAuth2/OpenID Connect:** 클라이언트 ID, 클라이언트 시크릿, 선택적으로 테넌트 ID(Azure), 인가·토큰·UserInfo URL 및 스코프 — Google, Microsoft, GitHub를 통한 로그인에 사용됩니다.

**연결 테스트 및 동기화:** LDAP/AD 제공자의 경우 "연결 테스트"는 발견된 사용자 수와 예시 사용자 이름을 표시합니다. 클라이언트 임포트가 활성화되어 있으면 "지금 동기화"가 수동 동기화를 실행합니다. 결과(처리됨/신규/업데이트됨/비활성화됨)는 실행 직후 페이지에 메시지로 표시됩니다.

## 알아두면 좋은 점

- 이 페이지는 **관리자만** 접근할 수 있으며, 슈퍼바이저 역할은 접근할 수 없습니다.
- Bind 비밀번호와 클라이언트 시크릿은 암호화되어 저장되며 화면에서는 마스킹(`***`)되어 표시됩니다. 변경 없이 저장하면 기존 값이 유지됩니다.
- 지속적인 동기화 로그(이력, 오류 이력)는 현재 화면에 표시되지 **않으며** — 가장 최근 수동 실행 결과만 표시됩니다.
- 디렉터리에서 더 이상 발견되지 않는 사람은 동기화 시 삭제되지 않고, 소속 종료일을 통해 비활성화됩니다. 나중에 다시 나타나면 자동으로 재활성화됩니다.
- 디렉터리에서 가져온 외부 식별자(LDAP ObjectGUID, 없으면 대체로 Distinguished Name)는 중복을 방지합니다. 재동기화 시 동일한 식별자가 발견되면 새로 생성하는 대신 기존 인물을 업데이트합니다.
