# 모델 연결 테스트

모든 언어 모델이 Klacksy에 적합한 것은 아닙니다 — Klacksy는 산문으로 답하는 대신 구조화된 함수 호출을 신뢰성 있게 제공해야 합니다. 설정 > Klacksy 모델 체크에서 Klacks는 구성된 모든 모델에 대해 이를 자동으로 테스트합니다.

## 작동 방식

**테스트:** "최적 모델 찾기"를 클릭하면 구성된 모든 모델(비활성화된 모델 포함)에 동일한 작은 테스트 과제가 전송됩니다: "Anna Müller라는 새 직원을 등록해줘." 모델은 텍스트 응답이 아니라 알맞은 구조화된 함수 호출과 올바른 이름 파라미터로 응답해야 합니다. 또한 Klacksy의 시스템 프롬프트와 기능이 들어갈 수 있도록 컨텍스트 윈도우가 최소 32,000 토큰이어야 합니다. 각 테스트는 30초의 시간 제한 안에서 실행됩니다.

**결과:** 두 기준을 모두 충족하는 모델은 "적합"으로 판정되어 자동으로 활성화되며, 부적합한 모델은 비활성화됩니다. 결과 목록은 적합성, 그다음 함수 호출 능력, 비용, 지연 시간, 컨텍스트 윈도우 순으로 정렬됩니다.

**기본 모델 지정:** "기본값으로 설정"을 통해 결과 목록에서 직접 Klacksy가 앞으로 기본으로 사용할 모델을 선택할 수 있습니다. 직접 선택하지 않으면 Klacks가 가장 적합한 모델을 자동으로 기본값으로 설정합니다.

## 알아두면 좋은 점

- 이 테스트는 등록된 모든 모델을 검사하며, 활성화된 모델만이 아닙니다 — 지금까지 비활성화되어 있던 모델도 테스트를 통과하면 활성화될 수 있습니다.
- 테스트 통과가 비용에 대한 안전장치는 아닙니다: 비싸지만 적합한 모델도 마찬가지로 활성화됩니다. 토큰 1000개당 비용은 결과 목록에서 확인할 수 있습니다.
- 이 테스트는 실행 능력을 위한 함수 호출에 특화되어 있으며, 순수 전사 정리를 위해 더 낮은 컨텍스트 윈도우로도 충분한 언어 설정의 모델 체크와는 별개입니다.
