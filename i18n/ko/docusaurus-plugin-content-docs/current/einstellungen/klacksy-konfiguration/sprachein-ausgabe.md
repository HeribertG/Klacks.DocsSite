# 음성 입출력

Klacksy는 음성으로도 조작할 수 있습니다 — 설정 > Klacksy 언어에서 사용자의 음성을 텍스트로 변환하는 서비스(Speech-to-Text)와 Klacksy가 응답할 때 사용하는 음성(Text-to-Speech)을 지정합니다.

## 작동 방식

**음성 인식(STT):** 네 가지 엔진 중에서 선택할 수 있습니다 — 무료인 브라우저(Web Speech API), Deepgram(실시간 스트리밍), Groq Whisper(빠르고 저렴함), AssemblyAI(스트리밍). 브라우저를 제외한 모든 경우에는 자체 API 키를 등록하고 연결을 직접 테스트할 수 있습니다.

**음성 출력(TTS):** 역시 네 개의 제공자가 있습니다 — Edge TTS(무료), OpenAI TTS, ElevenLabs, Google Cloud TTS. 선택한 제공자에 따라 Klacks가 사용 가능한 음성을 자동으로 불러오며, 여기서도 연결을 테스트할 수 있습니다.

**출력 모드:** 응답이 텍스트로만, 오디오로만, 수동으로 실행할 수 있는 음성 출력이 포함된 텍스트로, 또는 자동 음성 출력이 포함된 텍스트로 표시될지를 결정합니다. 무음 감지(500~3000ms, 기본값 1500ms)는 말이 끝난 후 녹음이 자동으로 전송되기까지 대기하는 시간을 결정합니다.

**전사 정리:** 이 옵션이 활성화되어 있으면 STT의 원본 텍스트가 추가로 LLM을 거쳐 불필요한 군더더기 표현을 제거하고, 자기 정정을 정리하고, 문법과 숫자를 다듬습니다. 이때 사용할 모델은 직접 선택하며, "최적 모델 찾기"를 통해 Klacks는 컨텍스트 윈도우가 16,000 토큰 이상인 활성화된 모든 모델을 테스트하여 가장 저렴하고 빠른 후보를 제안합니다(모델당 타임아웃 30초). 정리 프롬프트는 편집 가능하며 공장 초기 설정으로 되돌릴 수 있습니다.

**사전:** 음성 인식이 자주 잘못 인식하는 용어(예: 고유명사나 전문 용어)에 대해, 올바른 용어, 카테고리, 발음 변형, 선택적 언어 지정과 함께 항목을 등록할 수 있습니다 — 이후 Klacksy는 이러한 용어를 정확하게 교정합니다.

## 알아두면 좋은 점

- 브라우저 모드는 API 키가 필요 없지만, 해당 브라우저의 음성 인식 성능에 의존하며 품질 면에서 유료 서비스와 비교할 수 없습니다.
- API 키는 저장 후 "구성됨"으로만 표시되며, 다시는 평문으로 표시되지 않습니다.
- 여기서의 모델 체크는 전사 정리에 대한 적합성(컨텍스트 윈도우 ≥ 16,000 토큰)만 검사합니다 — Klacksy의 주 모델로서의 일반적인 적합성에는 별도의 더 엄격한 테스트가 있습니다(모델 연결 테스트 참조).
