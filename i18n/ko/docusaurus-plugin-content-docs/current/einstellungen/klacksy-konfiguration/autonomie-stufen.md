# 자율성 단계

Klacksy는 질문하기 전에 얼마나 독립적으로 행동해도 될까요? 설정 > Klacksy 자율성에서 이를 개인적으로 지정할 수 있습니다 — 이 설정은 사용자 계정별로 적용되며, 모든 사용자에게 공통으로 적용되지 않습니다.

## 작동 방식

**네 단계:**

- **제안** — Klacksy는 제안만 하며, 모든 변경 사항에는 사용자의 확인이 필요합니다.
- **보조** — 되돌릴 수 있는 작업은 바로 실행되고, 그 외 모든 작업은 확인이 필요합니다.
- **자율** (기본값) — 특히 민감한 작업을 제외한 모든 작업이 바로 실행됩니다.
- **완전 자율** — 여러 단계로 구성된 계획도 중간 승인 없이 진행됩니다.

**단계 분류가 작동하는 방식:** Klacksy가 수행할 수 있는 모든 작업은 내부적으로 위험도에 따라 분류되어 있습니다. 순수 읽기 작업은 항상 즉시 실행되고, 되돌릴 수 있거나 시나리오에 종속된 작업(예: 테스트 계획에서)은 최소 보조 단계가 필요하며, 되돌릴 수 없는 작업은 최소 자율 단계가 필요합니다. 민감한 작업 — 사용자 관리, 권한, 그리고 자율성 설정 자체의 변경 — 은 **선택된 단계와 무관하게** 항상 명시적인 확인을 요구합니다.

**확인 워크플로:** 작업에 사용자의 승인이 필요한 경우, Klacksy는 즉시 실행하지 않고 5분의 대기 시간과 함께 보류해 두며, 어떤 일이 일어날지 요약해 줍니다. 사용자가 자신의 표현으로 명시적으로 동의해야만 Klacksy는 보류된 작업을 실행합니다 — Klacksy는 결코 사용자를 대신해 스스로 확인하지 않습니다.

## 알아두면 좋은 점

- 자율성 단계는 사용자 계정별 개인 설정입니다 — 각 사용자는 자신에게 맞는 편안한 단계를 선택할 수 있습니다.
- 단계가 높다고 해서 아무것도 묻지 않는다는 뜻은 아닙니다: 민감한 작업은 언제나 확인이 필요합니다.
- 기본값은 자율 단계입니다 — 이는 자율성 단계가 도입되기 전 Klacksy가 보이던 동작과 동일합니다.
