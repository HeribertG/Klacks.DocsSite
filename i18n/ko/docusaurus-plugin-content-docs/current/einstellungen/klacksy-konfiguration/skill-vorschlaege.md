# Klacksy 스킬 제안

Klacksy는 자신의 실수로부터 배웁니다: 채팅에서 Klacksy가 잘못된 스킬을 선택했다고 정정하면, 시스템은 이 정정 내용을 기억합니다. 설정 > Klacksy 스킬 제안에서 관리자는 이로부터 어떤 스킬 설명 개선안이 제안되는지 확인하고 그 반영 여부를 결정합니다.

## 작동 방식

**제안이 나오는 방식:** Klacksy가 채팅에서 요청에 대해 잘못된 스킬을 선택하고 사용자가 이를 정정하면, 해당 채팅 시퀀스는 "정정됨"으로 표시됩니다. 내부 최적화 엔진이 이러한 정정된 시퀀스를 분석하여 스킬 설명에 대한 구체적인 텍스트 변경안을 도출합니다 — 목표는 다음번에 Klacksy가 해당 스킬을 올바르게 할당하는 것입니다.

**제안 생성:** "제안 생성" 버튼은 최근 정정된 채팅 시퀀스 30개에 대한 분석을 실행하며, 패턴이 발견되면 새로운 제안을 생성합니다.

**제안 카드에 표시되는 내용:** 영향을 받는 스킬과 변경된 필드, 변경 전/후 비교, 왜 이 변경이 타당한지에 대한 근거, 그리고 — 펼쳐볼 수 있는 형태로 — 이 제안을 유발한 정확한 사용자 요청 내용.

**결정:** 미결정 제안은 각각 개별적으로 **수락**(스킬 설명이 즉시 변경됨)되거나 **거부**(제안은 폐기되며 아무것도 변경되지 않음)됩니다. 자동 반영은 이루어지지 않습니다 — 항상 사람이 함께 결정합니다.

## 알아두면 좋은 점

- 채팅에서의 정정이 없으면 제안도 생성되지 않습니다 — 시스템은 단순한 사용이 아니라 실제 오류 정정 사례를 학습 기반으로 필요로 합니다.
- 이 기능은 Klacksy가 스킬을 올바르게 선택하도록 스킬의 설명 텍스트만 변경합니다 — 스킬이 실제로 수행하는 작업을 바꾸지는 않습니다.
- 이 기능은 관리자 권한이 필요합니다.
