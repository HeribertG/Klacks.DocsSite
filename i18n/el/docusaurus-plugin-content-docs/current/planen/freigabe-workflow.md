# Ροή έγκρισης: Από το προσχέδιο στο δεσμευτικό πλάνο

Ένα πρόγραμμα υπηρεσίας διέρχεται στο Klacks σαφώς καθορισμένα στάδια — ώστε να είναι ανά πάσα στιγμή ξεκάθαρο σε τι μπορεί να βασιστεί η ομάδα.

## Πώς λειτουργεί

Το πλάνο διατρέχει τέσσερα στάδια:

1. **Προσχέδιο** — η φάση εργασίας: όλα μπορούν ακόμη να αλλάξουν
2. **Επιβεβαιωμένο** — το πλάνο είναι έτοιμο ως προς το περιεχόμενο· αλλαγές πλέον μόνο ελεγχόμενα
3. **Εγκεκριμένο** — η αρμόδια αρχή έχει εγκρίνει
4. **Ολοκληρωμένο** — το πλάνο είναι σφραγισμένο και δεσμευτικό

Κάθε στάδιο **κλειδώνει την επεξεργασία** ανάλογα: Ό,τι έχει εγκριθεί, κανείς δεν το αλλάζει πλέον κατά λάθος με μεταφορά και απόθεση. Ποιος επιτρέπεται να ορίσει ποιο στάδιο είναι σαφώς καθορισμένο:

- **Την επιβεβαίωση** μπορεί να την κάνει κάθε συνδεδεμένο άτομο.
- **Η έγκριση** (και η ανάκλησή της) απαιτεί τον ρόλο **Supervisor** ή **Admin** — ξεχωριστά ανά ημέρα και ομάδα.
- **Η ολοκλήρωση** μιας περιόδου (και το μεταγενέστερο ξανάνοιγμα) επιφυλάσσεται **αποκλειστικά στον Admin**.

## Καλό να το ξέρετε

- Τα κλειδώματα ισχύουν παντού — ακόμη και ο [Αυτόματος προγραμματισμός: Το πλάνο σε δευτερόλεπτα](../planen/auto-planung.md) και ο [Klacksy](../ki/klacksy-ki-assistent.md) σέβονται την κατάσταση έγκρισης.
- Βραχυπρόθεσμες αλλαγές μετά την έγκριση (περιπτώσεις ασθένειας!) γίνονται ως συνειδητές, ανιχνεύσιμες διορθώσεις, όχι ως σιωπηρός επαναπρογραμματισμός.
- Για το λογιστικό κλείσιμο ενός μήνα υπάρχει επιπλέον το [Κλείσιμο περιόδου](../planen/periodenabschluss.md).
