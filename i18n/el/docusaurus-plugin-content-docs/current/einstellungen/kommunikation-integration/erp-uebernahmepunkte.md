Στις Ρυθμίσεις > Σημεία παραλαβής ERP διαμορφώνετε τον τρόπο με τον οποίο οι παραγγελίες από ένα εξωτερικό σύστημα ERP εισάγονται αυτόματα στο Klacks ως αρχεία XML.

## Πώς λειτουργεί

**Τρεις τρόποι παράδοσης:** Το σύστημα ERP μπορεί να τοποθετεί αρχεία στον χώρο αποθήκευσης του σημείου παραλαβής (το Klacks τα ανακτά αυτόματα σύμφωνα με το χρονοδιάγραμμα), να τα προωθεί απευθείας μέσω HTTP-POST σε ένα endpoint εισαγωγής (με ταυτοποίηση μέσω token πρόσβασης), ή μια διαχειρίστρια να ανεβάζει το αρχείο XML χειροκίνητα σε αυτή τη σελίδα με μεταφορά και απόθεση (drag-and-drop).

**Το σημείο παραλαβής (Drop-Point)** είναι το κεντρικό γραμματοκιβώτιο, διαχειριζόμενο από το ίδιο το Klacks, για όλα τα αρχεία εισαγωγής — εμφανίζει τρεις περιοχές: Εισερχόμενα (αναμονή για την επόμενη εκτέλεση), Επεξεργασμένα (αρχείο) και Σφάλματα (μη έγκυρα αρχεία μαζί με την αιτία σφάλματος, με δυνατότητα επανεισαγωγής). Ένα χρονοδιάγραμμα (έκφραση cron, προεπιλογή ωριαία) καθώς και ένας διακόπτης ενεργοποίησης για παύση ελέγχουν την αυτόματη ανάκτηση.

**Τα tokens πρόσβασης** χρειάζονται μόνο όταν ένα σύστημα ERP παραδίδει αρχεία μέσω push. Κάθε token ξεκινά με `klacks_erp_`, ισχύει για 1–730 ημέρες (προεπιλογή 365) και εμφανίζεται σε απλό κείμενο **μόνο μία φορά, κατά τη δημιουργία**. Ένα τέτοιο token ισχύει αποκλειστικά για το endpoint εισαγωγής, όχι για την υπόλοιπη πρόσβαση στο API, και μπορεί να ανακληθεί ανά πάσα στιγμή.

**Επεξεργασία:** Οι παραγγελίες αντιστοιχίζονται σε έναν υπάρχοντα ή νέο πελάτη βάσει του αναγνωριστικού συστήματος προέλευσης και της εξωτερικής αναφοράς πελάτη (ή, εναλλακτικά, εταιρείας/ταχυδρομικού κώδικα/οδού) και δημιουργούνται αρχικά ως πρόχειρο (draft), το οποίο οι υπεύθυνοι προγραμματισμού πρέπει να ελέγξουν και να σφραγίσουν. Μια τροποποιημένη νέα παράδοση μιας ήδη σφραγισμένης παραγγελίας ολοκληρώνει αυτόματα την παλιά και δημιουργεί μια νέα, συνδεδεμένη έκδοση· αμετάβλητες επαναληπτικές παραδόσεις δεν προκαλούν καμία ενέργεια.

## Καλό να το γνωρίζετε

- Εσφαλμένες μεμονωμένες παραγγελίες (π.χ. χωρίς ημέρα της εβδομάδας ή με διπλή αναφορά) απορρίπτονται, χωρίς να εμποδίζουν τις υπόλοιπες έγκυρες παραγγελίες του ίδιου αρχείου· ολόκληρο το αρχείο καταλήγει ωστόσο στην περιοχή σφαλμάτων, ώστε να μη διαφύγει τίποτα.
- Τα επεξεργασμένα αρχεία αρχειοθετούνται, δεν διαγράφονται.
- Η ακριβής μορφή XML (υποχρεωτικά πεδία, μορφές ημερομηνίας, λογική ημέρας εβδομάδας) τεκμηριώνεται στο εγχειρίδιο απευθείας στη σελίδα ρυθμίσεων, καθώς και μέσω ενός αρχείου παραδείγματος που μπορεί να ληφθεί.
- Αυτή η διαμόρφωση βρίσκεται στην περιοχή Ρυθμίσεων και επομένως προορίζεται αποκλειστικά για διαχειριστικούς ρόλους.
