# Πάροχοι LLM και μοντέλα

Ο Klacksy χρειάζεται ένα γλωσσικό μοντέλο για να κατανοεί και να ενεργεί. Στις Ρυθμίσεις > Πάροχοι LLM, Μοντέλα LLM και Ημερολόγιο συγχρονισμού LLM καθορίζετε ποιους παρόχους και ποια συγκεκριμένα μοντέλα επιτρέπεται να χρησιμοποιεί ο Klacks για αυτόν τον σκοπό.

## Πώς λειτουργεί

**Προσθήκη παρόχου:** Ένας πάροχος συγκεντρώνει τη βασική διεύθυνση URL, την έκδοση API και το κλειδί API ενός συγκεκριμένου φορέα. Έξι φορείς (OpenAI, Anthropic, Google, Azure, Mistral, DeepSeek) είναι ενσωματωμένοι απευθείας στον κώδικα· κάθε άλλος πάροχος συνδέεται μέσω μιας γενικής διεπαφής συμβατής με OpenAI — έτσι μπορούν να ενσωματωθούν π.χ. οι Groq, Together AI, Fireworks, OpenRouter, xAI, Cerebras, Perplexity ή ένα δικό σας φιλοξενούμενο μοντέλο. Μέσω της επιλογής «Αναζήτηση παρόχων» ο Klacks προτείνει επιπλέον φορείς που δεν έχετε ακόμη προσθέσει: είτε από έναν επιμελημένο κατάλογο επαληθευμένων διευθύνσεων, είτε — εφόσον έχει ρυθμιστεί αναζήτηση στο διαδίκτυο — μέσω διαδικτυακής έρευνας. Κάθε πρόταση ελέγχεται σε πραγματικό χρόνο ως προς τη διαθεσιμότητά της πριν εμφανιστεί· οι επιλεγμένοι πάροχοι προστίθενται απενεργοποιημένοι και χωρίς κλειδί API, το οποίο συμπληρώνετε εσείς στη συνέχεια.

**Κλειδί API και προτεραιότητα:** Μετά την αποθήκευση, το κλειδί δεν εμφανίζεται ποτέ ξανά σε απλό κείμενο στη διεπαφή — φαίνεται μόνο αν έχει καταχωριστεί κάποιο («Διαμορφωμένο»). Η προτεραιότητα καθορίζει τη σειρά με την οποία χρησιμοποιούνται οι ενεργοποιημένοι πάροχοι.

**Διαχείριση μοντέλων:** Στα Μοντέλα LLM δημιουργείτε ανά πάροχο μεμονωμένα μοντέλα (όνομα εμφάνισης, τεχνικό αναγνωριστικό μοντέλου στον φορέα, παράθυρο πλαισίου, μέγιστα tokens, κόστος ανά 1000 tokens εισόδου/εξόδου). Ένα μοντέλο μπορεί να επισημανθεί ως προεπιλεγμένο· ένας πάροχος δεν μπορεί να διαγραφεί όσο ένα από τα μοντέλα του αποτελεί την τρέχουσα προεπιλογή.

**Ημερολόγιο συγχρονισμού LLM:** Μια υπηρεσία παρασκηνίου ελέγχει αυτόματα, εξ ορισμού κάθε 24 ώρες, ποια μοντέλα των ενεργοποιημένων παρόχων έχουν γίνει πρόσφατα διαθέσιμα ή έχουν καταργηθεί. Τα νέα μοντέλα ελέγχονται εκ των προτέρων με ένα πραγματικό αίτημα δοκιμής και εισάγονται ενεργοποιημένα μόνο σε περίπτωση επιτυχίας· τα μοντέλα που δεν προσφέρονται πλέον απενεργοποιούνται αυτόματα. Το ημερολόγιο συγχρονισμού εμφανίζει το πλήρες ιστορικό ανά πάροχο με ημερομηνία, αριθμό νέων/απενεργοποιημένων/αποτυχημένων μοντέλων και αναπτυσσόμενα αποτελέσματα δοκιμών. Οι διαχειριστές λαμβάνουν επιπλέον, μετά την επόμενη σύνδεσή τους, μια ειδοποίηση toast για μη αναγνωσμένα συμβάντα συγχρονισμού.

## Καλό να το ξέρετε

- Μόνο έξι φορείς (OpenAI, Anthropic, Google, Azure, Mistral, DeepSeek) διαθέτουν αποκλειστική υλοποίηση — όλοι οι υπόλοιποι πάροχοι συνδέονται μέσω μιας γενικής διεπαφής συμβατής με OpenAI.
- Ο κατάλογος για την «Αναζήτηση παρόχων» περιλαμβάνει επί του παρόντος δώδεκα επαληθευμένους φορείς· προτάσεις από το διαδίκτυο εμφανίζονται μόνο αν έχει ρυθμιστεί αναζήτηση στο διαδίκτυο στις Ρυθμίσεις.
- Η υπηρεσία ημερολογίου συγχρονισμού λειτουργεί στο παρασκήνιο ανεξάρτητα από τη συνεδρία σας — δεν χρειάζεται να κρατάτε τη σελίδα ανοιχτή γι' αυτό.
