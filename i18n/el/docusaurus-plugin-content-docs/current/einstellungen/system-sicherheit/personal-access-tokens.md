# Personal Access Tokens

Τα Personal Access Tokens (PAT) είναι μακροχρόνια κλειδιά API, με τα οποία εξωτερικά εργαλεία και υπηρεσίες αποκτούν με ασφάλεια πρόσβαση στο Klacks χωρίς όνομα χρήστη και κωδικό πρόσβασης — για παράδειγμα βοηθοί τεχνητής νοημοσύνης όπως το Claude Desktop μέσω του Model Context Protocol (MCP), σενάρια αυτοματισμού ή διοχετεύσεις CI/CD.

## Πώς λειτουργεί

**Δημιουργία token:** Στο Ρυθμίσεις > Personal Access Tokens κάνετε κλικ στο "Δημιουργία νέου token", δίνετε ένα περιγραφικό όνομα (π.χ. `Claude Desktop` ή `CI-Pipeline`) και επιλέγετε διάρκεια ισχύος μεταξύ 1 και 730 ημερών (προεπιλογή: 365 ημέρες). Το token εμφανίζεται μετά τη δημιουργία με τη μορφή `klacks_pat_<τυχαία ακολουθία χαρακτήρων>` **ακριβώς μία φορά** — αντιγράψτε το αμέσως, καθώς μετά δεν μπορεί πλέον να ανακτηθεί.

**Χρήση:** Το token μεταδίδεται ως Bearer token στην κεφαλίδα `Authorization` κάθε αιτήματος API, π.χ. `Authorization: Bearer klacks_pat_...`. Για MCP clients όπως το Claude Desktop, το καταχωρείτε ως μεταβλητή περιβάλλοντος `AUTHORIZATION` του διακομιστή MCP που δείχνει στο `/mcp`.

**Διαχείριση:** Η επισκόπηση εμφανίζει όνομα, ημερομηνία δημιουργίας, ημερομηνία λήξης και τελευταία χρήση κάθε token — το ίδιο το token σε απλό κείμενο δεν εμφανίζεται ποτέ ξανά εκεί. Τα tokens που λήγουν τις επόμενες 30 ημέρες επισημαίνονται στη λίστα. Μέσω του κουμπιού Ανάκληση, ένα token κλειδώνεται αμέσως και οριστικά.

## Καλό είναι να γνωρίζετε

- Το Klacks αποθηκεύει μόνο ένα SHA-256 hash του token, ποτέ το ίδιο το token σε απλό κείμενο — ένα χαμένο token δεν μπορεί να ανακτηθεί, μόνο να ανακληθεί και να δημιουργηθεί εκ νέου.
- Κάθε token είναι συνδεδεμένο με τον λογαριασμό χρήστη σας και έχει τα ίδια δικαιώματα με τον δικό σας λογαριασμό — δεν υπάρχει λεπτομερής περιορισμός εμβέλειας (scope) ανά token.
- Χρησιμοποιήστε ξεχωριστό token για κάθε εργαλείο, ώστε σε περίπτωση υποψίας κατάχρησης να μπορείτε να το ανακαλέσετε στοχευμένα, χωρίς να διακόψετε άλλες ενσωματώσεις.
