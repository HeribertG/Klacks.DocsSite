# التحديثات

في القائمة الإعدادات > التحديثات، تشاهدون الإصدار الحالي المثبَّت وأحدث إصدار متاح من Klacks، وتتحكّمون في سلوك التحديث التلقائي، ويمكنكم عند الحاجة تشغيل تحديث أو التراجع عنه (Rollback) يدويًا.

## كيف يعمل الأمر

**الحالة:** تعرض البطاقة الإصدار الحالي، وما إذا كان هناك إصدار أحدث متاح، بالإضافة إلى عملية التحديث الجارية أو الأخيرة مع حالتها.

**تشغيل تحديث أو التراجع عنه:** عبر "التثبيت الآن" تبدأون تحديثًا متاحًا يدويًا؛ أما "التراجع" فيلغي آخر عملية تحديث ناجحة. كلا الإجراءين محظوران طالما أن هناك عملية جارية بالفعل.

**تهيئة الأتمتة:**
- **تفعيل التحديثات التلقائية** يُشغّل أو يُوقف الفحص والتنفيذ التلقائيين للتحديثات.
- **الإشعار فقط** لا يُطلق أي تحديث تلقائي، لكنه يُبلغ عند توفر إصدار جديد.
- **القناة** تختار بين `Stable` و`Beta` كمصدر للتحديثات.
- **فترة الفحص** (بالساعات) و**نافذة الصيانة** (وقت البدء/الانتهاء) تحدّدان وتيرة الفحص أو التثبيت والإطار الزمني الذي يتم ضمنه.
- **عدد النسخ الاحتياطية المُحتفَظ بها** يحدّد عدد النسخ الاحتياطية التي تُحفظ قبل الترحيل (Migration)، قبل أن تُحذف النسخ الأقدم تلقائيًا.

**السجل:** يسرد جدول آخر عمليات التحديث مع النوع والإصدار المستهدف والحالة ووقت الطلب.

## من المفيد معرفته

- ينشئ Klacks نسخة احتياطية تلقائيًا قبل كل عملية ترحيل — ولا يُطبَّق أي ترحيل دون نسخة احتياطية.
- إذا فشل فحص السلامة بعد تحديث ما، يتراجع Klacks تلقائيًا إلى الإصدار السابق (استعادة النسخة الاحتياطية ← تفعيل الإصدار القديم ← إعادة الفحص)؛ ولا يستلزم الأمر تدخلًا يدويًا إلا إذا فشل هذا التراجع نفسه.
- يستعيد التراجع اليدوي بشكل محدَّد النسخة الاحتياطية المرتبطة بآخر عملية ناجحة، ويُفعّل الإصدار الأصلي لتلك العملية — وليس مجرد "الرجوع بإصدار واحد إلى الوراء".
- التحديث والتراجع إجراءان مخصّصان للمسؤول (Admin) فقط.
