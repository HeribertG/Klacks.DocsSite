# 审批流程：从草稿到具有约束力的排班表

在 Klacks 中，排班表会经历几个清晰定义的阶段 — 这样团队随时都能明确知道，哪些安排是可以信赖的。

## 运作方式

排班表依次经过四个阶段：

1. **草稿** — 工作阶段：一切都还可以修改
2. **已确认** — 排班内容已经确定；改动只能以受控方式进行
3. **已批准** — 责任方已予以放行
4. **已完成** — 排班表已封存，正式生效

每个阶段都会相应地**锁定编辑权限**：已批准的内容，任何人都不会再通过拖放误改。谁有权设置哪个阶段，规则很明确：

- **确认**：任何已登录用户均可操作。
- **批准**（以及撤销批准）：需要 **Supervisor（主管）** 或 **Admin（管理员）** 角色 — 且按日期和分组分别授权。
- **结束**某个期间（以及之后重新开启）：**仅限 Admin（管理员）**。

## 小贴士

- 锁定在所有地方均有效 — [自动排班](../planen/auto-planung.md)和 [Klacksy](../ki/klacksy-ki-assistent.md) 同样会遵守审批状态。
- 审批之后的临时改动（比如有人生病！）会作为有意识、可追溯的更正处理，而不是悄无声息的重新排班。
- 若要对某个月份进行财务上的最终结算，还可以使用[期间结算](../planen/periodenabschluss.md)。
