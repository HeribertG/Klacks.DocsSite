# 模块化排班：分而治之

与其在一份庞杂的总排班表中疲于应对，不如将其拆分为清晰的小型子排班表 — Klacks 会自动保持整体同步。

## 运作方式

1. **拆分** — 将庞大的排班表划分为易于管理的区块：按团队、部门或地点。
2. **统一排班** — 在多个区块工作的员工只需排班一次，便会在各处自动正确显示。
3. **自动同步** — 任何改动都会立即在各处生效。

其中的诀窍在于：排班表保存的是**引用而非副本**。每个班次只存在一次 — 无论它出现在多少份子排班表中。这意味着重复排班无处遁形：一项持续运行的冲突检测会在改动被最终保存之前立即发现重叠。

## 小贴士

- 共享人员在其他区块的班次会在子排班表中显示为锁定条目 — 每位排班人员都能看到「自己的人」何时被别处占用，却无法改动他人的排班表。
- 子排班表的可见性通过基于分组的访问权限控制。
- 模块化排班与[自动排班](../planen/auto-planung.md)相辅相成：每个区块都可以单独进行自动排班。
