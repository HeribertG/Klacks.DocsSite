# Klacks 背后的理念

Klacks 是一款真正属于您的排班管理软件——而不是某个云服务商的。贯穿整个产品的,是三个始终如一的选择。

## 数据自主,而非云依赖

Klacks 运行在您自己的基础设施之上——数据不必离开您的机房。对于 Spitex 居家护理、医院或安保服务这类敏感行业来说,这绝非锦上添花的选项:把个人数据交给陌生的服务器、被供应商牢牢绑定,这些对它们而言往往根本行不通。Klacks 颠覆了这种主从关系:数据存放在哪里,由您决定,而不是由软件厂商决定。

## AI 自由选择,而非厂商锁定

内置的 AI 助手 Klacksy 并不依附于某一家 AI 厂商。无论是 OpenAI、Anthropic、Google、DeepSeek,还是本地部署的模型——成本、数据隐私和供应商的取舍,都由您来定,而不是由 Klacks 替您决定。这样一来,AI 能力和完整的数据自主权就不再是非此即彼的选择,而是可以兼得的。

## 开源透明,而非黑盒

Klacks 以 MIT 许可证开源——后端、前端和 Docker 镜像全部公开可查。您可以审查代码、fork 它、做自己的定制,或者参与到后续开发中来。这不是一句营销口号,而是可以自行验证的事实:全部源代码都公开在 GitHub 上。

## 适合谁

Klacks 面向那些认真对待上述三点的排班型企业——尤其是 Spitex 居家护理机构、医院、安保服务公司、清洁和物流企业。它们的共同点是:需要 24/7 覆盖、面对复杂的休息时间与资质要求,并且对数据保护有着一套普通云方案未必能满足的高标准。

## 这在实践中意味着什么

- 您可以在 [Playground](https://klacks-software.ch:7643) 中免费试用 Klacks,无需注册,也无需销售洽谈。
- 您可以通过 On-Premise 套件(Docker 镜像、安装程序、数据库、HTTPS、自动更新)将其部署在自己的服务器上运行。
- 您可以查看源代码并自行修改——即便您对它做了二次开发,Klacks 依然属于您。
