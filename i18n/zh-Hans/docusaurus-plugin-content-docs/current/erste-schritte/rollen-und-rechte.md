# 角色与权限

Klacks 为登录账号仅提供两种可分配的权限级别：**主管（Supervisor）**与**管理员（Admin）**。不存在第三种独立的技术角色 — 下文中「计划员」之类的说法描述的是**日常工作中的一种职能**，而不是额外的权限级别。

## 两种真正的角色

**主管**可以创建、编辑和删除地址、分组、合同、缺勤记录和班次，也可以在排班表中批准某一天或某个分组，并可以再次撤回该批准。而单纯确认个人的工作时间则不需要任何特殊角色 — 任何登录账号都可以做到。

**管理员**在主管的全部权限之外，还可以：访问所有设置（包括用户管理本身）、结算并重新开启周期，以及管理身份提供程序、报表和日历规则等受特别保护的区域。

角色分配通过用户管理（齿轮图标）中的下拉菜单完成，并立即生效，无需单独点击保存。

## 「计划员」是一种职能，而非角色

在日常工作中，通常由某个主管账号承担实际的排班工作 — 启动自动排班、在排班网格中通过拖放进行调整、将某天或某个分组提交审批。这并不是一个独立的权限级别，而只是主管账号本来就拥有的权限。

## 小贴士

- 当前登录的用户也会显示在用户管理中，但无法在那里编辑自己、更改自己的权限或删除自己。
- 永久结算或重新开启周期始终只保留给管理员 — 即使某个主管已经批准了相应的天数或分组。
- 登录账号不必与某个员工基础数据档案关联 — 用户管理与人员管理是两个独立的区域。
