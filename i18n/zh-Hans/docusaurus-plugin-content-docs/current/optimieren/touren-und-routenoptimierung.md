# 路线与巡回优化

对于外勤工作 — 居家护理巡访、安保巡逻、保洁场所、货物配送 — Klacks 不仅规划*由谁执行*，还规划*按什么顺序执行*。

## 运作方式

每个工作地点都登记有地址；因此 Klacks 掌握各点之间的路程。**蚁群算法**（一种在路线问题上久经考验的优化方法）会根据这些工作地址计算出最合理的行进顺序：

1. **录入地址** — 在工作地点中一次性登记
2. **优化顺序** — Klacks 确定最短且合理的行进顺序；绕路与空驶就此消失
3. **在地图上显示** — 完成的巡回以路线形式呈现在地图上，可选择驾车、骑行或步行

如果某次外勤登记了固定时间窗口（例如仅限上午），Klacks 会在巡回规划中加以考虑 — 但这并非硬性保证。

## 小贴士

- 在距离与行车时间的计算上，Klacks 可以接入外部服务 OpenRouteService（GPS 坐标会匿名化后传输）— 具体取决于实例的配置。
- 巡回优化是面向容器式班次的独立功能模块；一般排班的自动排班则独立评估覆盖率、公平性与规则合规性。
- 每条巡回都可以单独选择交通方式 — 城市骑行巡回与乡村驾车巡回并行不悖。
- 如果有人临时缺勤，Klacks 会立即显示谁既有资质又有空。
