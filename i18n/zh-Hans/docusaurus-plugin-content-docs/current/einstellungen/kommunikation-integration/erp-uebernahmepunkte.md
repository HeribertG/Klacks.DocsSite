在「设置 > ERP 导入接入点」中，您可以配置外部 ERP 系统的订单如何以 XML 文件的形式自动导入 Klacks。

## 运作方式

**三种投递方式：** ERP 系统可以将文件放入导入接入点的文件存储区（Klacks 按计划自动取件）；也可以通过 HTTP POST 直接推送到导入端点（使用访问令牌进行身份验证）；此外，管理员也可以在此页面通过拖放方式手动上传 XML 文件。

**导入接入点**是由 Klacks 自身管理的中央「收件箱」，用于接收所有导入文件 — 它分为三个区域：待处理（等待下一次运行）、已处理（归档）和错误（连同错误原因的无效文件，可重新导入）。一个计划任务（Cron 表达式，默认每小时一次）以及一个用于暂停的启用开关，共同控制自动取件。

**访问令牌**仅在 ERP 系统通过推送方式投递文件时才需要。每个令牌均以 `klacks_erp_` 开头，有效期为 1–730 天（默认 365 天），并且**仅在创建时**以明文显示一次。此类令牌仅对导入端点有效，不适用于其余的 API 访问，并且可随时被吊销。

**处理流程：** 订单会根据来源系统 ID 和外部客户参考号（若无则退而使用公司名/邮编/街道）匹配到已有客户或新建客户，并首先以草稿形式生成，需由排班人员审核并封存。已封存订单的一次改动后的重新投递，会自动关闭旧订单并创建一个与之关联的新版本；未发生变化的重复投递不会触发任何操作。

## 小贴士

- 单条有误的订单（例如缺少星期几或参考号重复）会被拒绝，但不会阻塞同一文件中其余有效订单的导入；不过整个文件仍会被归入错误区域，以确保不会被遗漏。
- 已处理的文件会被归档，而不是删除。
- 具体的 XML 格式（必填字段、日期格式、星期几逻辑）已在设置页面上直接以手册形式记录，另有可下载的示例文件。
- 此配置位于设置区域，因此仅限管理类角色使用。
