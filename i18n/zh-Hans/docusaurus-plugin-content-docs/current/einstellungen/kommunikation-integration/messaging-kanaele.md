在「设置 > 消息服务提供商」中，您可以接入外部聊天和短信渠道，让 Klacks（以及 Klacksy）能够直接联系员工和客户 — 作为电子邮件的替代方式或补充方式。

## 运作方式

**通用原则：** 所有渠道都通过同一个列表进行配置：内部名称、显示名称、提供商类型，以及因渠道而异的访问凭据（令牌、账户 ID、证书等），此外每个条目还有一个开/关开关和一个测试按钮，用于立即检查连接是否正常。消息最终会发送给谁，取决于在员工或客户「地址 > 联系方式」中保存的渠道专属标识（电话号码、用户名、用户 ID 等）— 您本人无需手动填写任何原始 ID。

**消息服务在 Klacks 中的用途：** 作为电子邮件之外的额外通知方式，例如用于排班表的提示消息或 Klacksy 发送的消息 — 具体取决于哪些渠道被启用，以及每个人保存了哪些联系方式。

**11 种渠道概览：**

| 渠道 | 发送/接收 | 特殊说明 |
|---|---|---|
| Slack | 发送 + 接收 | 机器人还必须通过 `/invite` 显式邀请进入每个频道。 |
| Telegram | 发送 + 接收 | 接收方必须事先自行通过 `/start` 激活该机器人。 |
| Signal | 仅发送 | 需要自行搭建 signal-cli 网桥（Docker），并使用一个通过二维码绑定的独立电话号码 — 没有官方云端 API。 |
| Microsoft Teams | 仅发送 | 每个频道需要一个 Power Automate Webhook；不支持 1:1 消息，多个 Teams 频道需要多个配置条目。 |
| SMS | 仅发送 | 电话号码需为 E.164 格式；试用账户（例如 Twilio）只能发送给事先验证过的号码，且目标国家必须已开通。 |
| 微信 | 仅发送 | 在中国以外注册为服务号（Official Account）较为困难；只能触达最近 48 小时内有过互动的关注者。 |
| KakaoTalk | 仅发送 | 访问令牌仅约 6 小时有效，需手动续期；接收方必须是该应用的「好友」。 |
| Line | 发送 + 接收 | 每月发送额度依套餐而定；接收方 ID 只有在收到对方发来的消息后才能获知。 |
| Threema | 仅发送 | 需要付费的 Threema Gateway 账户（预付费模式）；由于未使用端到端模式，无法接收消息。 |
| Viber | 发送 + 接收 | 机器人只能联系事先订阅过它的用户；Webhook 需要可公开访问的 HTTPS 地址。 |
| Zalo | 仅发送 | 访问令牌约 25 小时后过期，只能在互动窗口期内发送消息。 |

## 小贴士

- 每个渠道都配有专门、详尽的手册，逐步说明如何在对应服务商处完成配置 — 可在该服务商配置条目所在的设置页面上直接查阅。
- 多个渠道可以同时启用；实际使用哪一个，取决于对应人员保存了哪些联系方式。
- 对于所有使用限时访问令牌的渠道（例如 KakaoTalk、Zalo），必须在 Klacks 之外向服务商续期该令牌，并在 Klacks 中补录，否则该渠道的发送功能会中断。
