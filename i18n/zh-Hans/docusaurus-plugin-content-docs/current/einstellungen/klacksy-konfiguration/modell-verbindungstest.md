# 模型连接测试

并非每个语言模型都适合用作 Klacksy 的引擎 — 它必须能够可靠地返回结构化的函数调用，而不是以文字叙述的方式回答。您可以在「设置 > Klacksy 模型检测」中，让 Klacks 针对所有已配置的模型自动进行这项测试。

## 运作方式

**测试内容：** 点击「查找最佳模型」后，系统会向每个已配置的模型（包括已禁用的模型）发送同一个小测试任务：「创建一名叫 Anna Müller 的新员工。」模型必须以正确的结构化函数调用和准确的姓名参数作答 — 而不是以文字形式回答。此外，上下文窗口必须至少达到 32,000 个 Token，才能容纳 Klacksy 的系统提示词和各项能力。每次测试的时间限制为 30 秒。

**结果：** 同时满足两项条件的模型被判定为「合适」并自动启用；不合适的模型会被禁用。结果列表首先按适用性排序，其次依次按函数调用能力、成本、延迟和上下文窗口排序。

**设定默认模型：** 通过「设为默认」，您可以直接在结果列表中选择今后 Klacksy 默认使用的模型。如果您不主动选择，Klacks 会自动将最合适的模型设为默认。

## 小贴士

- 测试会检查所有已录入的模型，而不仅仅是已启用的模型 — 此前被禁用的模型如果通过测试，同样会被启用。
- 测试通过并不代表成本可控：昂贵但合适的模型同样会被启用。每 1000 个 Token 的成本可以在结果列表中查看。
- 该测试专门针对支持行动能力的函数调用而设计，与语音设置中的模型检测相互独立 — 后者对于纯粹的转录文本清理，允许使用更小的上下文窗口。
