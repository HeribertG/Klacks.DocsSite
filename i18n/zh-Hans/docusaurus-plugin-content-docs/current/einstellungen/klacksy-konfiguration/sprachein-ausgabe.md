# 语音输入与输出

Klacksy 也可以通过语音进行操作 — 您可以在「设置 > Klacksy 语音」中设定，使用哪种服务把您的声音转换为文字（语音转文字，STT），以及 Klacksy 用哪种声音来回答您（文字转语音，TTS）。

## 运作方式

**语音识别（STT）：** 可供选择的引擎共有四种 — 免费的浏览器方案（Web Speech API）、Deepgram（实时流式处理）、Groq Whisper（速度快、成本低）以及 AssemblyAI（流式处理）。除浏览器方案外，其余各项都需要您自行填入 API 密钥，并可以直接测试连接。

**语音合成（TTS）：** 同样有四家服务商可选 — Edge TTS（免费）、OpenAI TTS、ElevenLabs 和 Google Cloud TTS。根据所选服务商的不同，Klacks 会自动加载可用的语音；这里同样可以测试连接。

**输出模式：** 决定回答以纯文字、纯语音、可手动触发语音播放的文字，还是自动语音播放的文字形式呈现。静音检测（500–3000 毫秒，默认 1500 毫秒）决定说话结束后要等待多久才自动发送录音。

**转录文本清理：** 启用该选项后，原始的 STT 文本会额外经过一个大语言模型处理，去除口头禅、消解自我纠正、并润色语法和数字表达。用于此目的的模型由您自行选择；通过「查找最佳模型」，Klacks 会测试所有已启用的、上下文窗口至少 16,000 个 Token 的模型，并推荐其中最便宜、最快的候选（每个模型超时时间为 30 秒）。清理提示词可以编辑，也可以重置为出厂设置。

**词典：** 对于语音识别经常识别错误的词语（例如专有名词或专业术语），您可以创建词条，包含正确的词语、类别、发音变体以及可选的语言归属 — Klacksy 之后会针对性地纠正这些词语。

## 小贴士

- 浏览器模式不需要 API 密钥，但依赖于所用浏览器自身的语音识别能力，质量无法与付费服务相比。
- API 密钥保存后只会显示为「已配置」，不会再以明文显示。
- 此处的模型检测只判断是否适合用于转录文本清理（上下文窗口 ≥ 16,000 个 Token）— 若要判断是否整体适合作为 Klacksy 的主模型，则有专门更严格的测试（参见「模型连接测试」）。
