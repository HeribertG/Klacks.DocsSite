# LLM 提供商与模型

Klacksy 需要一个语言模型才能理解和行动。您可以在「设置 > LLM 提供商」「LLM 模型」和「LLM 同步日志」中设定 Klacks 可以使用哪些服务商及具体哪些模型。

## 运作方式

**添加提供商：** 一个提供商整合了某个服务商的基础 URL、API 版本和 API 密钥。六家服务商（OpenAI、Anthropic、Google、Azure、Mistral、DeepSeek）已在代码中固定实现；其余所有提供商都通过一个通用的、兼容 OpenAI 接口的方式接入 — 借此可以接入例如 Groq、Together AI、Fireworks、OpenRouter、xAI、Cerebras、Perplexity 或自行部署的模型。通过「搜索提供商」，Klacks 还会为您推荐尚未添加的服务商：来源可以是经过整理的已验证地址目录，或者 — 如果已配置网络搜索功能 — 通过互联网检索获得。每个建议在展示前都会实时检测其可达性；被选中的提供商会以禁用状态、且不带 API 密钥的方式创建，之后需要您自行补充密钥。

**API 密钥与优先级：** 保存后，密钥在界面中不会再以明文显示 — 只会显示是否已配置密钥（「已配置」）。优先级决定了已启用的提供商被调用的先后顺序。

**管理模型：** 在「LLM 模型」中，您可以为每个提供商分别添加模型（显示名称、服务商处的技术模型 ID、上下文窗口、最大 Token 数、每 1000 个输入/输出 Token 的成本）。一个模型可以被标记为默认模型；只要某个提供商下的某个模型仍是当前默认模型，该提供商就无法被删除。

**LLM 同步日志：** 后台服务默认每 24 小时自动检查一次已启用的提供商有哪些模型新增或下线。新模型会先通过一次真实的测试请求进行验证，只有成功后才会以启用状态加入；不再提供的模型会被自动禁用。同步日志按提供商展示完整历史记录，包括日期、新增/禁用/失败模型的数量，以及可展开查看的测试结果。管理员在下次登录后还会收到一条关于未读同步事件的提示通知。

## 小贴士

- 只有六家服务商（OpenAI、Anthropic、Google、Azure、Mistral、DeepSeek）拥有专门的实现 — 其余所有提供商都通过通用的、兼容 OpenAI 接口的方式运行。
- 「搜索提供商」的目录目前包含十二家已验证的服务商；只有在设置中配置了网络搜索功能时，才会显示来自网络的建议。
- 同步日志服务在后台独立于您的会话运行 — 您无需为此保持页面处于打开状态。
