# 更新

在「设置 > 更新」中，您可以查看当前安装的版本和最新可用的 Klacks 版本，控制自动更新行为，并在需要时手动触发更新或回滚。

## 运作方式

**状态：** 卡片显示当前版本、是否有更新版本可用，以及正在进行或最近一次执行的更新操作及其状态。

**触发更新或回滚：** 点击「立即安装」可手动启动可用的更新；「撤销」会撤销上一次成功的更新操作。只要已有操作正在进行，这两个操作都会被锁定。

**配置自动化：**
- **启用自动更新** 用于开启或关闭自动检查和执行更新。
- **仅通知** 不会触发自动更新，但会在有新版本可用时提示您。
- **渠道** 用于在 `Stable`（稳定版）和 `Beta`（测试版）之间选择更新来源。
- **检查间隔**（小时）和**维护窗口**（起止时间）用于设定检查和安装的频率及时间范围。
- **保留备份数量** 决定在迁移前保留多少个备份，超出的旧备份会被自动删除。

**历史记录：** 表格列出最近的更新操作，包括类型、目标版本、状态和请求时间。

## 小贴士

- 每次迁移之前，Klacks 都会自动创建备份 — 没有备份就不会执行迁移。
- 如果更新后的健康检查失败，Klacks 会自动回滚到之前的版本（恢复备份 → 启用旧版本 → 再次检查）；只有当这个回滚过程本身失败时，才需要人工介入。
- 手动回滚会精确恢复与上一次成功操作对应的备份，并启用该操作的起始版本 — 而不是简单地「回退一个版本」。
- 更新和回滚仅限管理员操作。
