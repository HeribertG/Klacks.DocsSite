# 用户管理

在「设置 > 用户管理」中，您管理的是人员登录 Klacks 所使用的**登录账户**（系统用户）— 这与员工或地址管理不同，因为一个登录账户可以、但不是必须与某个人员的基础数据档案关联。

## 运作方式

**添加用户：** 「+ 添加用户」对话框会根据名字和姓氏（各至少 2 个字符）自动生成一个用户名建议。电子邮件地址为必填项，且必须包含 @ 符号；只有当所有字段都填写有效后才能保存。创建时会自动生成一个密码 — 若要让本人自行设置密码，请在创建后于该用户所在行使用「重置密码」。

**编辑用户：** 双击姓名所在行会打开编辑对话框，您可以在其中修改用户名 — 创建时生成的用户名只是一个建议值。

**重置密码：** 用户所在行中的「重置密码」链接会向其登记的邮箱地址发送一封邮件，本人可通过该邮件自行设置新密码。

**删除用户：** 垃圾桶按钮在删除对话框中确认后，会永久删除该登录账户。

**分配权限：** 通过该行中的下拉菜单分配**主管（Supervisor）**或**管理员（Admin）**角色；更改会立即生效，无需单独点击保存。这两个角色具体拥有哪些权限，详见「新手入门」中的「角色与权限」— 这里只涉及账户本身的管理。

## 小贴士

- 当前登录的用户也会显示在列表中，但无法在此编辑自己的信息、更改自己的权限或删除自己。
- 登录账户独立于人员/员工管理 — 例如您可以为一家外部 IT 公司创建账户，而无需为此建立员工档案。
- 管理员无法直接设置密码：新账户会获得一个自动生成的密码，之后的任何修改都需要通过「重置密码」的邮件链接完成。
