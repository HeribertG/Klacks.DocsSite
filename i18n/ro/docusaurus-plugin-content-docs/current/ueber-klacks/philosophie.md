# Filozofia din spatele Klacks

Klacks este planificarea personalului care vă aparține — nu unui furnizor de cloud. Trei decizii se regăsesc în întregul produs.

## Suveranitatea datelor în locul dependenței de cloud

Klacks rulează pe infrastructura dumneavoastră proprie — niciun set de date nu trebuie să părăsească sediul. Mai ales pentru domenii sensibile precum Spitex, spitale sau servicii de securitate, acest lucru nu este un simplu plus: datele personale pe servere străine și dependența de furnizor nu sunt adesea pur și simplu o opțiune pentru aceste organizații. Klacks inversează relația: dumneavoastră decideți unde se află datele, nu producătorul software-ului.

## Libertate de alegere a AI-ului în locul dependenței de furnizor

Klacksy, asistentul AI integrat, nu este legat de un singur furnizor de AI. Fie OpenAI, Anthropic, Google, DeepSeek sau un model rulat local — dumneavoastră decideți asupra costurilor, protecției datelor și furnizorului, nu Klacks. Astfel, suportul AI și suveranitatea deplină a datelor nu se exclud reciproc, ci merg împreună.

## Open source în locul unei cutii negre

Klacks este cu sursă deschisă sub licența MIT — backend-ul, frontend-ul și imaginile Docker pot fi consultate public. Puteți verifica codul, îl puteți face fork, puteți face propriile adaptări sau puteți contribui la dezvoltarea sa ulterioară. Aceasta nu este o promisiune de marketing, ci verificabilă: întregul cod sursă se află public pe GitHub.

## Pentru cine este făcut

Klacks se adresează organizațiilor cu planificare pe ture și intervenții care iau în serios aceste trei aspecte — în special organizațiilor Spitex, spitalelor, serviciilor de securitate, firmelor de curățenie și logistică. Numitorul comun: acoperire 24/7, cerințe complexe privind perioadele de repaus și calificările, precum și un standard de protecție a datelor pe care o soluție cloud interschimbabilă nu îl poate îndeplini întotdeauna.

## Ce înseamnă asta în practică

- Puteți testa Klacks gratuit în [Playground](https://klacks-software.ch:7643), fără înregistrare sau discuție de vânzare.
- Îl puteți opera cu un pachet On-Premise (imagini Docker, program de instalare, bază de date, HTTPS, actualizări automate) pe propriul server.
- Puteți consulta codul sursă și face modificări proprii — Klacks rămâne al dumneavoastră, chiar dacă îl dezvoltați în continuare.
