# การเพิ่มประสิทธิภาพเส้นทางและทัวร์งาน

สำหรับงานเคลื่อนที่ — ทัวร์เยี่ยมบ้านของ Spitex การตรวจตรารักษาความปลอดภัย งานทำความสะอาดสถานที่ หรืองานจัดส่ง — Klacks ไม่ได้วางแผนแค่ *ใคร* เท่านั้น แต่ยังวางแผน *ตามลำดับใด* ด้วย

## วิธีการทำงาน

สถานที่ปฏิบัติงานแต่ละแห่งจะถูกบันทึกพร้อมที่อยู่ ดังนั้น Klacks จึงรู้เส้นทางทั้งหมด **อัลกอริทึมมด (Ant Algorithm)** ซึ่งเป็นวิธีการเพิ่มประสิทธิภาพที่ได้รับการพิสูจน์แล้วสำหรับปัญหาด้านเส้นทาง จะคำนวณลำดับที่เหมาะสมที่สุดจากที่อยู่ของจุดปฏิบัติงาน:

1. **บันทึกที่อยู่** — กรอกไว้ที่สถานที่ปฏิบัติงานเพียงครั้งเดียว
2. **เพิ่มประสิทธิภาพลำดับ** — Klacks กำหนดลำดับที่สั้นและสมเหตุสมผลที่สุด ทางอ้อมและการเดินทางเปล่าจะหมดไป
3. **แสดงบนแผนที่** — ทัวร์ที่เสร็จสมบูรณ์จะปรากฏเป็นเส้นทาง เลือกได้ทั้งสำหรับรถยนต์ จักรยาน หรือเดินเท้า

หากมีการกำหนดช่วงเวลาที่แน่นอนสำหรับงานหนึ่ง ๆ (เช่น เฉพาะช่วงเช้า) ข้อมูลนี้จะถูกนำไปพิจารณาในการวางแผนทัวร์ด้วย — แต่ไม่ได้เป็นการรับประกันที่แน่นอนว่าจะปฏิบัติตามได้เสมอไป

## ข้อมูลที่ควรทราบ

- สำหรับการคำนวณระยะทางและเวลาเดินทาง Klacks สามารถเชื่อมต่อกับบริการภายนอก OpenRouteService ได้ (พิกัด GPS จะถูกส่งแบบไม่ระบุตัวตน) ขึ้นอยู่กับการตั้งค่าของแต่ละอินสแตนซ์
- การเพิ่มประสิทธิภาพทัวร์เป็นส่วนประกอบที่แยกออกมาต่างหากสำหรับกะงานแบบ Container ส่วนการวางแผนอัตโนมัติทั่วไปของตารางเวรจะประเมินความครอบคลุม ความเป็นธรรม และการปฏิบัติตามกฎอย่างเป็นอิสระจากส่วนนี้
- เลือกยานพาหนะได้แยกตามแต่ละทัวร์ — สะดวกสำหรับทัวร์จักรยานในเมืองควบคู่ไปกับทัวร์รถยนต์ในชนบท
- หากมีใครขาดงานกะทันหัน Klacks จะแสดงทันทีว่าใครมีคุณสมบัติและพร้อมปฏิบัติงาน
