# ปรัชญาเบื้องหลัง Klacks

Klacks คือระบบวางแผนกำลังคนที่เป็นของคุณ — ไม่ใช่ของผู้ให้บริการคลาวด์ มีการตัดสินใจสามประการที่ปรากฏอยู่ทั่วทั้งผลิตภัณฑ์

## อธิปไตยด้านข้อมูล แทนการพึ่งพาคลาวด์

Klacks ทำงานบนโครงสร้างพื้นฐานของคุณเอง — ไม่มีชุดข้อมูลใดที่จำเป็นต้องออกจากองค์กรของคุณ สำหรับอุตสาหกรรมที่มีความอ่อนไหวสูง เช่น Spitex โรงพยาบาล หรือหน่วยรักษาความปลอดภัย นี่ไม่ใช่เพียงสิ่งที่ดีต่อมีเท่านั้น: ข้อมูลส่วนบุคคลบนเซิร์ฟเวอร์ของผู้อื่นและการพึ่งพาผู้ให้บริการ มักไม่ใช่ทางเลือกที่ยอมรับได้เลยสำหรับองค์กรเหล่านี้ Klacks พลิกความสัมพันธ์นี้กลับด้าน: คุณเป็นผู้ตัดสินใจว่าข้อมูลของคุณจะอยู่ที่ใด ไม่ใช่ผู้ผลิตซอฟต์แวร์

## อิสระในการเลือก AI แทนการผูกติดกับผู้ให้บริการ

Klacksy ผู้ช่วย AI ที่ฝังอยู่ในตัว ไม่ได้ผูกติดกับผู้ให้บริการ AI รายใดรายหนึ่ง ไม่ว่าจะเป็น OpenAI, Anthropic, Google, DeepSeek หรือโมเดลที่รันในเครื่องของคุณเอง — คุณเป็นผู้ตัดสินใจเรื่องค่าใช้จ่าย ความเป็นส่วนตัว และผู้ให้บริการ ไม่ใช่ Klacks การสนับสนุนด้วย AI และอธิปไตยด้านข้อมูลเต็มรูปแบบจึงไม่ขัดแย้งกัน แต่กลับไปด้วยกันได้

## โอเพนซอร์ส แทนกล่องดำ

Klacks เป็นซอฟต์แวร์โอเพนซอร์สภายใต้สัญญาอนุญาต MIT — ทั้ง Backend, Frontend และ Docker Images สามารถเข้าดูได้แบบสาธารณะ คุณสามารถตรวจสอบโค้ด ทำการ Fork ปรับแต่งเอง หรือร่วมพัฒนาต่อได้ นี่ไม่ใช่คำสัญญาทางการตลาด แต่เป็นสิ่งที่ตรวจสอบได้จริง: ซอร์สโค้ดทั้งหมดเปิดเผยต่อสาธารณะบน GitHub

## เหมาะสำหรับใคร

Klacks เหมาะสำหรับองค์กรที่มีการวางแผนกะงานและการปฏิบัติงานที่ให้ความสำคัญกับสามประเด็นนี้อย่างจริงจัง — โดยเฉพาะองค์กร Spitex โรงพยาบาล หน่วยรักษาความปลอดภัย ธุรกิจทำความสะอาดและโลจิสติกส์ จุดร่วมของธุรกิจเหล่านี้คือ: การให้บริการตลอด 24/7 ข้อกำหนดด้านเวลาพักและคุณสมบัติที่ซับซ้อน และมาตรฐานด้านความเป็นส่วนตัวที่โซลูชันคลาวด์ทั่วไปไม่สามารถตอบสนองได้เสมอไป

## สิ่งนี้หมายความว่าอย่างไรในทางปฏิบัติ

- คุณสามารถทดลองใช้ Klacks ได้ฟรีใน [Playground](https://klacks-software.ch:7643) โดยไม่ต้องลงทะเบียนหรือพูดคุยกับฝ่ายขาย
- คุณสามารถติดตั้งใช้งานบนเซิร์ฟเวอร์ของคุณเองได้ด้วยแพ็กเกจ On-Premise (Docker Images, ตัวติดตั้ง, ฐานข้อมูล, HTTPS, การอัปเดตอัตโนมัติ)
- คุณสามารถเข้าดูซอร์สโค้ดและทำการเปลี่ยนแปลงได้ด้วยตนเอง — Klacks ยังคงเป็นของคุณ แม้ว่าคุณจะพัฒนาต่อยอดมันเองก็ตาม
