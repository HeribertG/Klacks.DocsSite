# การรับ-ส่งเสียง

Klacksy สามารถใช้งานผ่านเสียงพูดได้เช่นกัน — ที่ การตั้งค่า > เสียงของ Klacksy คุณตั้งค่าได้ว่าจะใช้บริการใดในการแปลงเสียงพูดของคุณเป็นข้อความ (Speech-to-Text) และจะใช้เสียงใดในการตอบกลับของ Klacksy (Text-to-Speech)

## วิธีการทำงาน

**การรู้จำเสียงพูด (STT):** มีให้เลือกสี่เอนจิน คือ เบราว์เซอร์แบบไม่มีค่าใช้จ่าย (Web Speech API), Deepgram (สตรีมมิงแบบเรียลไทม์), Groq Whisper (รวดเร็วและราคาย่อมเยา) และ AssemblyAI (สตรีมมิง) สำหรับทุกตัวเลือกยกเว้นเบราว์เซอร์ คุณต้องกรอกคีย์ API ของคุณเองและสามารถทดสอบการเชื่อมต่อได้ทันที

**การสังเคราะห์เสียง (TTS):** มีผู้ให้บริการสี่รายเช่นกัน คือ Edge TTS (ไม่มีค่าใช้จ่าย), OpenAI TTS, ElevenLabs และ Google Cloud TTS ขึ้นอยู่กับผู้ให้บริการที่เลือก Klacks จะโหลดเสียงที่มีให้เลือกใช้โดยอัตโนมัติ และสามารถทดสอบการเชื่อมต่อได้เช่นกัน

**โหมดการตอบกลับ:** กำหนดว่าคำตอบจะแสดงเป็นข้อความเท่านั้น เสียงเท่านั้น ข้อความพร้อมปุ่มเปิดเสียงด้วยตนเอง หรือข้อความพร้อมเสียงพูดอัตโนมัติ การตรวจจับความเงียบ (500–3000 มิลลิวินาที ค่าเริ่มต้น 1500 มิลลิวินาที) กำหนดว่าจะรอนานเท่าใดหลังจากพูดจบ ก่อนที่การบันทึกจะถูกส่งออกไปโดยอัตโนมัติ

**การปรับแต่งข้อความถอดเสียง:** หากเปิดใช้งานตัวเลือกนี้ ข้อความ STT ดิบจะถูกส่งผ่าน LLM เพิ่มเติม เพื่อลบคำฟุ่มเฟือย แก้ไขการพูดซ้ำแก้ตัวเอง และปรับไวยากรณ์/ตัวเลขให้เรียบร้อยขึ้น คุณเลือกโมเดลที่ใช้สำหรับขั้นตอนนี้ได้เอง ผ่านปุ่ม "ค้นหาโมเดลที่ดีที่สุด" Klacks จะทดสอบโมเดลที่เปิดใช้งานทั้งหมดที่มีหน้าต่างบริบทตั้งแต่ 16,000 โทเคนขึ้นไป และเสนอตัวเลือกที่ถูกที่สุดและเร็วที่สุด (เวลาทดสอบจำกัด 30 วินาทีต่อโมเดล) พรอมป์สำหรับการปรับแต่งสามารถแก้ไขได้และรีเซ็ตกลับเป็นค่าเริ่มต้นจากโรงงานได้

**พจนานุกรม:** สำหรับคำที่การรู้จำเสียงพูดมักเข้าใจผิดเป็นประจำ (เช่น ชื่อเฉพาะหรือศัพท์เทคนิค) คุณสามารถเพิ่มรายการที่มีคำที่ถูกต้อง หมวดหมู่ รูปแบบการออกเสียงต่าง ๆ และภาษาที่เกี่ยวข้อง (ถ้ามี) — Klacksy จะแก้ไขคำเหล่านี้ให้ถูกต้องโดยเฉพาะ

## ข้อมูลที่ควรทราบ

- โหมดเบราว์เซอร์ไม่ต้องใช้คีย์ API แต่ขึ้นอยู่กับความสามารถในการรู้จำเสียงพูดของเบราว์เซอร์แต่ละตัว และคุณภาพเทียบไม่ได้กับบริการแบบเสียค่าใช้จ่าย
- คีย์ API หลังจากบันทึกแล้วจะแสดงเพียงสถานะ "กำหนดค่าแล้ว" เท่านั้น ไม่แสดงเป็นข้อความธรรมดาอีกเลย
- การตรวจสอบโมเดลในที่นี้จะตรวจสอบเฉพาะความเหมาะสมสำหรับการปรับแต่งข้อความถอดเสียงเท่านั้น (หน้าต่างบริบท ≥ 16,000 โทเคน) — สำหรับความเหมาะสมโดยรวมในฐานะโมเดลหลักของ Klacksy มีการทดสอบเฉพาะที่เข้มงวดกว่าแยกต่างหาก (ดูหน้าการทดสอบการเชื่อมต่อโมเดล)
