ที่การตั้งค่า > Identity Provider คุณสามารถเชื่อมต่อ Klacks กับบริการไดเรกทอรีภายนอก — LDAP/Active Directory หรือ OAuth2/OpenID Connect — เพื่อนำเข้าข้อมูลพนักงานโดยอัตโนมัติ และ/หรือใช้สำหรับการเข้าสู่ระบบ

## วิธีการทำงาน

**LDAP/Active Directory:** คุณกรอกข้อมูล Host, Port (389/636), SSL, Base DN, Bind DN, รหัสผ่าน Bind และตัวกรองผู้ใช้ (User Filter) สวิตช์ "ใช้สำหรับนำเข้าลูกค้า" จะเปิดใช้งานการนำเข้าอัตโนมัติ: นามสกุล ชื่อ คำนำหน้า บริษัท และที่อยู่ จะถูกดึงมาจากแอตทริบิวต์ในไดเรกทอรี — สำหรับ OpenLDAP และ Active Directory บางแอตทริบิวต์มีชื่อต่างกัน (เช่น `uid` เทียบกับ `sAMAccountName`, `street` เทียบกับ `streetAddress`) การจับคู่ฟิลด์เหล่านี้ถูกกำหนดไว้ตายตัวในแบ็กเอนด์และไม่สามารถแก้ไขได้ผ่านหน้าจอ

**OAuth2/OpenID Connect:** Client-ID, Client-Secret, Tenant-ID (Azure) แบบเลือกได้, URL สำหรับ Authorization, Token และ UserInfo รวมถึง Scopes — สำหรับการเข้าสู่ระบบผ่าน Google, Microsoft หรือ GitHub

**ทดสอบและซิงค์การเชื่อมต่อ:** สำหรับผู้ให้บริการ LDAP/AD ปุ่ม "ทดสอบการเชื่อมต่อ" จะแสดงจำนวนผู้ใช้ที่พบ พร้อมตัวอย่างชื่อผู้ใช้ หากเปิดใช้งานการนำเข้าลูกค้าไว้ ปุ่ม "ซิงค์ตอนนี้" จะสั่งให้ทำการซิงค์ด้วยตนเอง ผลลัพธ์ (ประมวลผลแล้ว/ใหม่/อัปเดตแล้ว/ปิดใช้งานแล้ว) จะปรากฏเป็นข้อความบนหน้านั้นทันทีหลังจากนั้น

## ข้อมูลที่ควรทราบ

- หน้านี้เข้าถึงได้ **เฉพาะผู้ดูแลระบบ (Administrator)** เท่านั้น ไม่รวมบทบาท Supervisor
- รหัสผ่าน Bind และ Client-Secret จะถูกจัดเก็บแบบเข้ารหัส และแสดงในหน้าจอแบบปิดบัง (`***`) เท่านั้น หากบันทึกโดยไม่มีการเปลี่ยนแปลง ค่าเดิมจะยังคงอยู่
- ปัจจุบันหน้าจอ **ไม่มี** บันทึกการซิงค์แบบถาวร (ประวัติ, ประวัติข้อผิดพลาด) — มีเพียงผลลัพธ์ของการซิงค์ด้วยตนเองครั้งล่าสุดเท่านั้น
- บุคคลที่ไม่พบในไดเรกทอรีอีกต่อไปจะไม่ถูกลบระหว่างการซิงค์ แต่จะถูกปิดใช้งานผ่านวันที่สิ้นสุดการเป็นสมาชิก หากพวกเขากลับมาปรากฏอีกครั้งในภายหลัง ระบบจะเปิดใช้งานให้อัตโนมัติ
- รหัสประจำตัวจากภายนอก (LDAP ObjectGUID หรือ Distinguished Name เป็นทางเลือกสำรอง) ช่วยป้องกันข้อมูลซ้ำซ้อน: หากพบรหัสเดียวกันในการซิงค์ครั้งถัดไป บุคคลที่มีอยู่เดิมจะถูกอัปเดตแทนที่จะสร้างใหม่
