ที่ การตั้งค่า > DeepL คุณกรอกคีย์ API ของ DeepL ที่ Klacks จะใช้แปลข้อความโดยอัตโนมัติ — ปัจจุบันใช้เป็นหลักสำหรับอีเมลขาเข้าในกล่องข้อความ

## วิธีการทำงาน

**คีย์ API:** มีช่องกรอกเพียงช่องเดียวสำหรับคีย์ API ของ DeepL (รูปแบบ `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx:fx` สำหรับ Free-API ที่ไม่มีค่าใช้จ่าย หรือไม่มีส่วนต่อท้าย `:fx` สำหรับ Pro-API แบบเสียค่าใช้จ่าย) Klacks จะตรวจจับจากส่วนต่อท้าย `:fx` โดยอัตโนมัติว่าจะเรียกใช้ที่อยู่ API ใดของ DeepL — ไม่จำเป็นต้องสลับระหว่าง Free และ Pro แยกต่างหาก คุณสามารถสมัครบัญชีและรับคีย์ได้ที่ deepl.com/pro-api

**การใช้งาน:** ใน "กล่องข้อความ" อีเมลแต่ละฉบับสามารถแปลเป็นภาษาที่ตั้งค่าไว้ในอินเทอร์เฟซปัจจุบันได้ผ่านไอคอนแปลภาษา ต้นฉบับและคำแปลสามารถเปรียบเทียบกันได้ผ่านปุ่มสลับ หากไม่มีคีย์ที่กำหนดไว้ ฟังก์ชันนี้จะไม่สามารถใช้งานได้

## ข้อมูลที่ควรทราบ

- การตั้งค่านี้เป็นการตั้งค่าส่วนกลางสำหรับทั้งระบบ — ไม่มีคีย์แยกต่อผู้ใช้แต่ละคน
- ไม่มีปุ่ม "ทดสอบ" แยกต่างหากในหน้านี้ ว่าคีย์ใช้งานได้หรือไม่จะทราบได้จากการลองแปลครั้งแรกในกล่องข้อความ
- หากคีย์ไม่ถูกต้องหรือหมดอายุ Klacks จะตรวจจับได้จากการตอบกลับของ DeepL (ข้อผิดพลาดการยืนยันตัวตน) และแจ้งเตือนเรื่องนี้ แทนที่จะแสดงคำแปลที่ว่างเปล่าหรือผิดพลาด
- คีย์จะถูกจัดเก็บแบบเข้ารหัสและแสดงในฟอร์มแบบซ่อนบางส่วน
