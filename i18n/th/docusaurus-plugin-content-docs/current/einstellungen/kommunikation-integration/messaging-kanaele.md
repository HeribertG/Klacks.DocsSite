ที่การตั้งค่า > ผู้ให้บริการ Messaging คุณสามารถเชื่อมต่อช่องทางแชทและ SMS ภายนอก ซึ่ง Klacks (และ Klacksy) จะใช้ในการติดต่อพนักงานและลูกค้าโดยตรง — แทนที่หรือเสริมจากอีเมล

## วิธีการทำงาน

**หลักการร่วมกัน:** ทุกช่องทางถูกตั้งค่าผ่านรายการเดียวกัน ได้แก่ ชื่อภายใน ชื่อที่แสดง ประเภทผู้ให้บริการ และข้อมูลการเข้าถึงที่แตกต่างกันไปตามช่องทาง (Token, Account-ID, ใบรับรอง ฯลฯ) พร้อมสวิตช์เปิด/ปิด และปุ่มทดสอบสำหรับแต่ละรายการ เพื่อตรวจสอบการเชื่อมต่อได้ทันที ผู้รับข้อความจะถูกระบุจากรหัสประจำช่องทาง (หมายเลขโทรศัพท์ ชื่อผู้ใช้ User-ID ฯลฯ) ที่บันทึกไว้ที่พนักงานหรือลูกค้าภายใต้ ที่อยู่ > ข้อมูลติดต่อ — คุณไม่จำเป็นต้องกรอกรหัสดิบด้วยตนเอง

**การใช้งาน Messaging ใน Klacks:** เป็นช่องทางแจ้งเตือนเพิ่มเติมนอกเหนือจากอีเมล เช่น สำหรับการแจ้งเตือนจากตารางเวรหรือข้อความจาก Klacksy — ขึ้นอยู่กับว่าเปิดใช้งานช่องทางใดและบันทึกไว้กับบุคคลนั้นหรือไม่

**11 ช่องทางโดยสรุป:**

| ช่องทาง | ส่ง/รับ | ลักษณะพิเศษ |
|---|---|---|
| Slack | ส่ง + รับ | ต้องเชิญบอทเข้าแต่ละช่องทางด้วย `/invite` เพิ่มเติมด้วยตนเอง |
| Telegram | ส่ง + รับ | ผู้รับต้องเปิดใช้งานบอทด้วยคำสั่ง `/start` ด้วยตนเองก่อน |
| Signal | ส่งอย่างเดียว | ต้องใช้ signal-cli-bridge ที่ดูแลเอง (Docker) พร้อมหมายเลขโทรศัพท์เฉพาะที่จับคู่ผ่าน QR-Code — ไม่มี Cloud-API อย่างเป็นทางการ |
| Microsoft Teams | ส่งอย่างเดียว | ใช้ Power-Automate-Webhook หนึ่งรายการต่อหนึ่งช่องทาง ไม่รองรับข้อความแบบ 1:1 หากต้องการหลายช่องทาง Teams ต้องสร้างหลายรายการ |
| SMS | ส่งอย่างเดียว | หมายเลขโทรศัพท์ในรูปแบบ E.164 สำหรับบัญชีทดลอง (เช่น Twilio) จะส่งได้เฉพาะหมายเลขที่ยืนยันไว้ล่วงหน้า และประเทศปลายทางต้องได้รับการเปิดใช้งาน |
| WeChat | ส่งอย่างเดียว | การยืนยันตัวตนเป็น Official Account นอกประเทศจีนค่อนข้างยาก และเข้าถึงได้เฉพาะผู้ติดตามที่มีปฏิสัมพันธ์ในช่วง 48 ชั่วโมงที่ผ่านมา |
| KakaoTalk | ส่งอย่างเดียว | Access Token ใช้งานได้ประมาณ 6 ชั่วโมงเท่านั้นและต้องต่ออายุด้วยตนเอง ผู้รับต้องเป็น "เพื่อน" กับแอปก่อน |
| Line | ส่ง + รับ | โควตาการส่งรายเดือนจำกัดตามแผนที่ใช้ รหัสผู้รับจะทราบได้ก็ต่อเมื่อมีข้อความเข้ามาก่อน |
| Threema | ส่งอย่างเดียว | ต้องมีบัญชี Threema-Gateway แบบเสียเงิน (ระบบเติมเงิน) ไม่รองรับการรับข้อความ เนื่องจากไม่ได้ใช้โหมด End-to-End |
| Viber | ส่ง + รับ | บอทสามารถส่งข้อความได้เฉพาะผู้ที่สมัครติดตามไว้ก่อนแล้วเท่านั้น Webhook ต้องเข้าถึงได้ผ่าน HTTPS สาธารณะ |
| Zalo | ส่งอย่างเดียว | Access Token หมดอายุหลังประมาณ 25 ชั่วโมง และส่งข้อความได้เฉพาะภายในช่วงเวลาที่มีการโต้ตอบเท่านั้น |

## ข้อมูลที่ควรทราบ

- แต่ละช่องทางมีคู่มือฉบับสมบูรณ์พร้อมขั้นตอนการตั้งค่ากับผู้ให้บริการนั้น ๆ — เปิดดูได้โดยตรงจากหน้าการตั้งค่าของรายการผู้ให้บริการนั้น
- สามารถเปิดใช้งานหลายช่องทางพร้อมกันได้ ส่วนช่องทางที่ถูกใช้จริงขึ้นอยู่กับข้อมูลติดต่อที่บันทึกไว้กับบุคคลนั้น ๆ
- สำหรับทุกช่องทางที่มี Access Token แบบมีอายุจำกัด (เช่น KakaoTalk, Zalo) ต้องต่ออายุนอกระบบ Klacks ที่ผู้ให้บริการ แล้วนำมากรอกใน Klacks ใหม่ มิฉะนั้นการส่งผ่านช่องทางนั้นจะหยุดทำงาน
