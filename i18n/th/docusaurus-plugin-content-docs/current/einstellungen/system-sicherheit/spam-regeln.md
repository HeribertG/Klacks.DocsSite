# กฎกรองสแปม

ในเมนู การตั้งค่า > กฎกรองสแปม คุณสามารถกำหนดกฎการกรองที่ใช้จัดประเภทอีเมลขาเข้าในโมดูล Inbox ว่าเป็นสแปมโดยอัตโนมัติได้ — กฎเหล่านี้มีผลทั้งในขณะดึงอีเมลใหม่ตามปกติ (IMAP-Polling) และเมื่อมีการประเมินอีเมลที่รับเข้ามาแล้วใหม่อีกครั้งในภายหลัง

## วิธีการทำงาน

**สร้างกฎใหม่:** เลือกประเภทของกฎ — ผู้ส่งมีข้อความว่า, โดเมนผู้ส่ง, หัวข้อมีข้อความว่า หรือเนื้อหามีข้อความว่า — จากนั้นกรอกรูปแบบที่ต้องการตรวจสอบ แล้วคลิกเพิ่ม

**จัดการกฎ:** แต่ละกฎสามารถเปิดหรือปิดใช้งานได้ผ่านช่องกาเครื่องหมาย โดยไม่ต้องลบทิ้ง หรือจะลบออกอย่างถาวรผ่านปุ่มถังขยะก็ได้

## ข้อมูลที่ควรทราบ

- กฎที่ถูกปิดใช้งานจะยังคงถูกบันทึกไว้ แต่จะไม่มีผลต่ออีเมลใหม่หรืออีเมลที่ประเมินใหม่อีกต่อไป — มีประโยชน์สำหรับการพักการใช้กฎชั่วคราวเพื่อทดสอบ แทนที่จะลบทิ้งไปเลย
- Klacks ใช้กล่องขาเข้าเดียวกันสำหรับการนำเข้าคำสั่งซื้อ ERP และโมดูล Inbox อีเมลทั่วไป (สถาปัตยกรรมกล่องจดหมายเดียว) ดังนั้นกฎที่กำหนดไว้กว้างเกินไป (เช่น ทั้งโดเมนผู้ส่ง) อาจทำให้อีเมลคำสั่งซื้อ ERP ที่เข้ามาถูกจัดเป็นสแปมไปด้วยในทางทฤษฎี — ควรตรวจสอบกฎใหม่อย่างรอบคอบก่อนเปิดใช้งาน
- ใช้กฎแบบโดเมนผู้ส่งเพื่อบล็อกองค์กรผู้ส่งที่ไม่ต้องการทั้งหมด แทนที่จะต้องบันทึกทีละที่อยู่ด้วยกฎแบบผู้ส่งมีข้อความว่า
