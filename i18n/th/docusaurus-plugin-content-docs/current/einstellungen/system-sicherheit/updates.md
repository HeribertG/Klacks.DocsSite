# การอัปเดต

ในเมนูการตั้งค่า > การอัปเดต คุณจะเห็นเวอร์ชันของ Klacks ที่ติดตั้งอยู่ในปัจจุบันและเวอร์ชันล่าสุดที่มีให้ใช้งาน สามารถควบคุมพฤติกรรมการอัปเดตอัตโนมัติ และหากจำเป็นก็สามารถสั่งอัปเดตหรือย้อนกลับเวอร์ชันด้วยตนเองได้

## วิธีการทำงาน

**สถานะ:** การ์ดนี้แสดงเวอร์ชันปัจจุบัน ว่ามีเวอร์ชันใหม่กว่าหรือไม่ รวมถึงการอัปเดตที่กำลังทำงานอยู่หรือครั้งล่าสุดพร้อมสถานะ

**สั่งอัปเดตหรือย้อนกลับ:** ปุ่ม "ติดตั้งตอนนี้" ใช้เริ่มการอัปเดตที่มีอยู่ด้วยตนเอง ส่วน "เลิกทำ" จะย้อนกลับการอัปเดตที่สำเร็จครั้งล่าสุด ทั้งสองปุ่มจะถูกล็อกไว้ตราบใดที่มีการดำเนินการอื่นทำงานอยู่แล้ว

**ตั้งค่าระบบอัตโนมัติ:**
- **เปิดใช้งานการอัปเดตอัตโนมัติ** เปิดหรือปิดการตรวจสอบและดำเนินการอัปเดตอัตโนมัติ
- **แจ้งเตือนเท่านั้น** จะไม่สั่งอัปเดตอัตโนมัติ แต่จะแจ้งเมื่อมีเวอร์ชันใหม่ให้ใช้งาน
- **ช่องทาง** เลือกระหว่าง `Stable` และ `Beta` เป็นแหล่งที่มาของการอัปเดต
- **ช่วงเวลาตรวจสอบ** (เป็นชั่วโมง) และ **ช่วงเวลาบำรุงรักษา** (เวลาเริ่ม/สิ้นสุด) กำหนดว่าจะตรวจสอบและติดตั้งบ่อยแค่ไหนและในช่วงเวลาใด
- **จำนวนแบ็กอัปที่เก็บไว้** กำหนดว่าจะเก็บแบ็กอัปไว้กี่ชุดก่อนการย้ายข้อมูล ก่อนที่แบ็กอัปเก่าจะถูกลบโดยอัตโนมัติ

**ประวัติ:** ตารางแสดงรายการการอัปเดตล่าสุด พร้อมประเภท เวอร์ชันเป้าหมาย สถานะ และเวลาที่ร้องขอ

## ข้อมูลที่ควรทราบ

- ก่อนการย้ายข้อมูลทุกครั้ง Klacks จะสร้างแบ็กอัปโดยอัตโนมัติ — หากไม่มีแบ็กอัป จะไม่มีการย้ายข้อมูลเกิดขึ้น
- หากการตรวจสอบความสมบูรณ์หลังการอัปเดตล้มเหลว Klacks จะย้อนกลับไปยังเวอร์ชันก่อนหน้าโดยอัตโนมัติ (กู้คืนแบ็กอัป → เปิดใช้งานเวอร์ชันเก่า → ตรวจสอบอีกครั้ง) จะต้องดำเนินการด้วยตนเองก็ต่อเมื่อการย้อนกลับนี้เองล้มเหลว
- การย้อนกลับด้วยตนเองจะกู้คืนแบ็กอัปที่เกี่ยวข้องกับการดำเนินการที่สำเร็จครั้งล่าสุดโดยเฉพาะ และเปิดใช้งานเวอร์ชันต้นทางของแบ็กอัปนั้น ไม่ใช่แค่ "ย้อนกลับหนึ่งเวอร์ชัน"
- การอัปเดตและการย้อนกลับเป็นการดำเนินการที่สงวนไว้สำหรับผู้ดูแลระบบ (Admin) เท่านั้น
